測量誤差緩解¶

簡介¶

測量校準是用來減輕測量誤差。 主要的想法是準備所有 :math:」2^n」 基底輸入狀態並計算其他基底狀態下測量數的概率。 從這些校準中，可以更正另一個感興趣的實驗的平均結果。 此記事本提供範例，說明如何使用 ignis.mitigation.measurement 模組。

[1]:

# Import general libraries (needed for functions)
import numpy as np
import time

# Import Qiskit classes
import qiskit
from qiskit import QuantumRegister, QuantumCircuit, ClassicalRegister, Aer
from qiskit.providers.aer import noise
from qiskit.tools.visualization import plot_histogram

# Import measurement calibration functions
from qiskit.ignis.mitigation.measurement import (complete_meas_cal, tensored_meas_cal,
                                                 CompleteMeasFitter, TensoredMeasFitter)

3 量子位元範例（校準矩陣）¶

假設我們要為五量子位元暫存器 [Q0,Q1,Q2,Q3,Q4] 中的三個量子位元Q2、Q3 和 Q4 生成校準矩陣。

由於我們有 3 個量子位元，所以有 \(2^3=8\) 種可能的量子態。

產生測量校準電路¶

首先，我們會為完全希爾伯特空間生成測量校準電路的清單。 每個電路都會建立一個基底狀態。 如果有 \(n=3\) 個量子位元，則我們可以得到 \(2^3=8\) 校準電路。

下列函數 complete_meas_cal 會傳回包含校準電路的 QuantumCircuit 物件清單**meas_calibs**，以及校準狀態標籤的序列 state_labels 。

此函數的輸入可以用下列三種格式的其中一種方式來提供：

  • qubit_list: 是要對以下執行測量更正的量子位元清單，或：

  • qr (QuantumRegister): 一個量子暫存器，或：

  • cr (ClassicalRegister): 一個經典暫存器。

此外，我們還可以提供一個字串 circlabel，它會被新增在電路名稱的開頭，以進行唯一識別。

例如，在此情況中，輸入是一個包含量子位元Q2, Q3和Q4的五量子位元的 QuantumRegister:

[2]:

# Generate the calibration circuits
qr = qiskit.QuantumRegister(5)
qubit_list = [2,3,4]
meas_calibs, state_labels = complete_meas_cal(qubit_list=qubit_list, qr=qr, circlabel='mcal')

列印 \(2^3=8\) 狀態標籤（對應於其3量子位元Q2,Q3,Q4）：

修正對三量子位元 GHZ 狀態的量測噪音¶

作為範例，我們以三個量子位元 Q2、Q3 和 Q4 的 GHZ 狀態開始:

\[\mid GHZ \rangle = \frac{\mid{000} \rangle + \mid{111} \rangle}{\sqrt{2}}\]
[11]:

# Make a 3Q GHZ state
cr = ClassicalRegister(3)
ghz = QuantumCircuit(qr, cr)
ghz.h(qr[2])
ghz.cx(qr[2], qr[3])
ghz.cx(qr[3], qr[4])
ghz.measure(qr[2],cr[0])
ghz.measure(qr[3],cr[1])
ghz.measure(qr[4],cr[2])

[11]:

<qiskit.circuit.instructionset.InstructionSet at 0x7f869dce3040>

我們現在執行校準電路（以上方的噪音模型）：

[12]:

job = qiskit.execute([ghz], backend=backend, shots=5000, noise_model=noise_model)
results = job.result()

我們現在計算沒有任何錯誤緩解和有錯誤緩解的結果，也就是在將校準矩陣套用到結果之後。

套用校準時有兩種適合的方法（如果未定義方法，則會使用 『least_squares』）。 - ‘pseudo_inverse’，這是校準矩陣的直接反轉， - **』least_squares』 **，它提供了額外的限制使最後獲得的是有物理意義的概率。

要更正的原始資料，可以用多種形式提供：

  • 形式1：來自 results.get_counts 的計數字典，

  • 形式2：一個 length=len(state_labels) 的計數清單，

  • 形式3 ：一個 length=M*len(state_labels) 的計數清單，其中 M 是一個整數（例如，與斷層掃描資料一起使用），

  • 形式4： 一個 qiskit 結果（例如上述的結果）。

[13]:

# Results without mitigation
raw_counts = results.get_counts()

# Get the filter object
meas_filter = meas_fitter.filter

# Results with mitigation
mitigated_results = meas_filter.apply(results)
mitigated_counts = mitigated_results.get_counts(0)

張量化的緩解¶

如果其誤差為本端(local)，則其校準可被簡化。 “local error” 的意思是，其誤差可以被張量化到該量子位元集合的子集。 在此情況下，對校準矩陣的計算只需要小於 \(2^n\) 個狀態。

假設誤差只本端地(locally) 作用在量子位元 2 以及量子位元 3 和 4 的配對。 使用函數 tensored_meas_cal 以建構校準電路。不像之前，我們需要將其量子位元清單明確地分割成子集區域。

[20]:

# Generate the calibration circuits
qr = qiskit.QuantumRegister(5)
mit_pattern = [[2],[3,4]]
meas_calibs, state_labels = tensored_meas_cal(mit_pattern=mit_pattern, qr=qr, circlabel='mcal')

我們現在檢索所產生電路的名稱。 請注意，在每一個標籤（長度為 3）中，最不重要的位元對應於量子位元2 ，中間位元對應於量子位元3 ，且最重要的位元對應於量子位元 4。

[21]:

for circ in meas_calibs:
    print(circ.name)

mcalcal_000
mcalcal_010
mcalcal_101
mcalcal_111

讓我們詳細說明電路名稱。 我們看到只有 4 個電路，而不是 8 個。 必要電路的總數是 \(2^m\) ，其中 \(m\) 是目標子集中的 量子位元數目（這裡為 \(m=2\)）。

量子位元 3 和 4 的每個基底態確切地出現一次。量子位元 2 只需要兩個基底態，所以這些基底態平均分配到四個實驗。 例如，量子位元 2 的狀態 『0』 出現在狀態標籤 『000』 及 『010’中。

我們現在使用與之前相同的噪音模型在 Aer 模擬器上執行校準電路。事實上，這噪音其實是分別地局限於量子位元 3 和 4，但假設我們還不知道，而且我們只知道它是量子位元 2 的局部噪音。

使用測量錯誤緩解以執行 Qiskit 演算法¶

若要在 Qiskit 演算法中執行量子電路時使用測量錯誤緩解，我們需要在 QuantumInstance 中包含各自的測量錯誤擬合實例。 此物件也會保留所選後端的規格。

以下，我們用 VQE 搜尋哈密頓算符的基態的例子，說明 Aqua 演算法的測量錯誤緩解。

首先，我們需要匯入提供後端 (backend) 的程式庫 (libraries)，以及執行演算法所需要的類別 (classes)。

[33]:

# Import qiskit functions and libraries
from qiskit import Aer, IBMQ
from qiskit.circuit.library import TwoLocal
from qiskit.utils import QuantumInstance
from qiskit.algorithms import VQE
from qiskit.algorithms.optimizers import COBYLA
from qiskit.opflow import X, Y, Z, I, CX, T, H, S, PrimitiveOp
from qiskit.providers.aer import noise

# Import error mitigation functions
from qiskit.ignis.mitigation.measurement import CompleteMeasFitter

然後，我們初始化運行算法所需的實例 (instances)。

[34]:

# Initialize Hamiltonian
h_op =  (-1.0523732 * I^I) + \
                (0.39793742 * I^Z) + \
                (-0.3979374 * Z^I) + \
                (-0.0112801 * Z^Z) + \
                (0.18093119 * X^X)
# Initialize trial state
ansatz = TwoLocal(h_op.num_qubits, ['ry', 'rz'], 'cz', reps=3, entanglement='full')
# Initialize optimizer
optimizer = COBYLA(maxiter=350)

在這裡，我們選擇 Aer 的 qasm_simulator` 作為後端 (backend) 並添加自定義噪音模型。 IBMQ 所提供的實際量子後端的應用可以程式碼中評論找到。

[35]:

# Generate a noise model
noise_model = noise.NoiseModel()
for qi in range(h_op.num_qubits):
    read_err = noise.errors.readout_error.ReadoutError([[0.8, 0.2],[0.1,0.9]])
    noise_model.add_readout_error(read_err, [qi])

# Initialize the backend configuration using measurement error mitigation with a QuantumInstance
qi_noise_model_qasm = QuantumInstance(backend=Aer.get_backend('qasm_simulator'), noise_model=noise_model, shots=1000,
                                      measurement_error_mitigation_cls=CompleteMeasFitter,
                                      measurement_error_mitigation_shots=1000)

# Intialize your TOKEN and provider with

# Initialize algorithm to find the ground state
vqe = VQE(ansatz, optimizer, quantum_instance=qi_noise_model_qasm)

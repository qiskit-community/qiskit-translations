Qiskit Aer: Simulação de pulso de um modelo de backend¶

Este caderno mostra como usar o simulador de pulso Aer usando um modelo gerado a partir de um backend. Em particular, realizamos um experimento Rabi para encontrar uma amplitude de \(\pi\)-pulse amplitude em um modelo de um qubit no backend Armonk, gerado a partir do backend de simulação FakeArmonk.

Índice¶

  1. Importações

  2. Construa modelo a partir do backend <#model> __

  3. Execute experimentos Rabi e encaixe em :math:pi`-pulse amplitude <#rabi>`__

1. Importações¶

Importar bibliotecas gerais:

[1]:

import numpy as np

Importar o gerador de experimentos Rabi e fitter do Ignis e outras funções para a submissão de tarefas:

[2]:

from qiskit.ignis.characterization.calibrations import rabi_schedules, RabiFitter

from qiskit.pulse import DriveChannel
from qiskit.compiler import assemble
from qiskit.qobj.utils import MeasLevel, MeasReturnType

Importar PulseSimulator e PulseSystemModel para simulação de pulso, bem como o backend de imitação Armonk:

[3]:

# The pulse simulator
from qiskit.providers.aer import PulseSimulator

# Object for representing physical models
from qiskit.providers.aer.pulse import PulseSystemModel

# Mock Armonk backend
from qiskit.test.mock.backends.armonk.fake_armonk import FakeArmonk

2. Construir modelo a partir do backend¶

Esta seção demonstra o uso da função PulseSystemModel.from_backend para gerar objetos PulseSystemModel a partir de um backend.

Nota: Parâmetros do Hamiltoniano relatados nos backends mudam ao longo do tempo. Para estabilizar este tutorial em relação a essas mudanças, após instanciar FakeArmonk, substituímos manualmente o Hamiltoniano relatado na configuração do backend para valores estáticos.

Instanciar o backend simulado:

[4]:

armonk_backend = FakeArmonk()

Sobrescreva manualmente os parâmetros do Hamiltoniano:

[5]:

freq_est = 4.97e9
drive_est = 6.35e7
armonk_backend.defaults().qubit_freq_est = [freq_est]
armonk_backend.configuration().hamiltonian['h_str']= ['wq0*0.5*(I0-Z0)', 'omegad0*X0||D0']
armonk_backend.configuration().hamiltonian['vars'] = {'wq0': 2 * np.pi * freq_est, 'omegad0': drive_est}
armonk_backend.configuration().hamiltonian['qub'] = {'0': 2}
armonk_backend.configuration().dt = 2.2222222222222221e-10

Gerar modelo a partir do backend:

[6]:

armonk_model = PulseSystemModel.from_backend(armonk_backend)

3. Executar experimentos Rabi e ajuste de \(\pi\)-pulse amplitude¶

Em seguida, executamos experimentos Rabi gerados usando Ignis no simulador usando o modelo de sistema gerado a partir do backend Armonk.

Primeiro, construa schedules de experimento Rabi:

[7]:

# qubit list
qubits = [0]

# drive amplitudes to use
num_exps = 64
drive_amps = np.linspace(0, 1.0, num_exps)

# drive shape parameters
drive_duration = 2048
drive_sigma = 256

# list of drive channels
drive_channels = [DriveChannel(0)]

# construct the schedules
rabi_schedules, xdata = rabi_schedules(amp_list=drive_amps,
                                       qubits=qubits,
                                       pulse_width=drive_duration,
                                       pulse_sigma=drive_sigma,
                                       drives=drive_channels,
                                       inst_map=armonk_backend.defaults().instruction_schedule_map,
                                       meas_map=armonk_backend.configuration().meas_map)

Monte o qobj para o envio da tarefa. Ao montar schedules de pulso a serem usados com o simulador de pulso, passe o PulseSimulator como o backend.

[8]:

backend_sim = PulseSimulator()

rabi_qobj = assemble(rabi_schedules,
                     backend=backend_sim,
                     meas_level=1,
                     meas_return='avg',
                     shots=512)

Execute a simulação:

[9]:

sim_result = backend_sim.run(rabi_qobj, armonk_model).result()

Gere o diagrama de oscilação Rabi e encontre a amplitude do \(\pi\)-pulse:

[10]:

rabi_fit = RabiFitter(sim_result, xdata, qubits, fit_p0 = [1.5, 2, 0, 0])

# get the pi amplitude
pi_amp = rabi_fit.pi_amplitude(0)

# plot
rabi_fit.plot(0)
print('Pi Amp: %f'%pi_amp)

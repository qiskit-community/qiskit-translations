Construindo Instruções Pulse¶

Programas Pulse, que são chamados de Schedules, descrevem sequências de instruções para a eletrônica de controle.

Nesta página, cobriremos, em profundidade, estas Instructions disponíveis por meio do Qiskit Pulse:

  • `Delay(duration: int, channel) <#delay>`__

  • `Play(pulse, channel) <#play>`__

  • `SetFrequency(frequency, channel) <#frequency>`__

  • `ShiftPhase(phase, channel) <#phase>`__

  • `Acquire(duration, channel, mem_slot, reg_slot) <#acquire>`__

Cada tipo de instrução possui seu próprio conjunto de operandos. Como você pode ver, acima, cada um deles inclui, pelo menos, um Channel para especificar, onde a instrução será aplicada.

Canais são rótulos de linhas de sinal do hardware de controle para o chip quântico.

  • DriveChannels são tipicamente utilizados para conduzir rotações de um único qubit,

  • ControlChannels são tipicamente usados para portas multi-qubit ou linhas de transmissão adicionais para qubits ajustáveis,

  • MeasureChannels são específicos para transmissão de pulsos, que estimulam a leitura, e

  • AcquireChannels são usados para acionar digitalizadores, que coletam sinais de leitura.

DriveChannels, ControlChannels, e MeasureChannels são todos PulseChannels; Isto significa que eles suportam a transmissão de pulsos, enquanto o AcquireChannel é apenas um canal receptor e não pode gerar ondas.

Para os seguintes exemplos, vamos criar uma instância DriveChannel para cada Instruction, que aceita um PulseChannel. Canais recebem um argumento index inteiro. Exceto pelos ControlChannels, o índice mapeia trivialmente para o rótulo do qubit.

[1]:

from qiskit.pulse import DriveChannel

channel = DriveChannel(0)

## Delay

Uma das instruções mais simples, que podemos construir, é a Delay. Trata-se de uma instrução de bloqueio, que diz à eletrônica de controle, para não produzir nenhum sinal, no canal determinado pela duração especificada. Ela é útil para controlar a cronometragem (timing) de outras instruções.

A duração aqui e, em qualquer outro lugar, é em termos do tempo de ciclo do serviço (1 / taxa de amostragem), dt.

Para construir uma instrução Delay, passamos a duração e o canal:

[2]:

from qiskit.pulse import Delay

delay_5dt = Delay(5, channel)

onde channel pode ser qualquer tipo de canal, incluindo o AcquireChannel.

Isto é tudo que há pra fazer. Esta instrução, delay_5dt, está pronta para ser incluída em uma Schedule. Qualquer instrução anexada após delay_5dt no mesmo canal executará cinco intervalos de tempo mais tarde do que teria sem esse atraso.

## Play

A instrução Play é responsável por executar pulsos. É simples construir uma:

play = Play(pulse, channel)

Vamos esclarecer o que é o argumento pulse e explorar algumas maneiras diferentes de construir um.

Pulsos¶

Um Pulse especifica um envelope de pulso arbitrário. A frequência de modulação e a fase da onda de saída são controladas pelas instruções SetFrequency e ShiftPhase, que abordaremos, em seguida.

A imagem abaixo pode fornecer alguma intuição para o motivo pelo qual elas são especificadas, separadamente. Pense nos pulsos, que descrevem seus envelopes como entrada, para um gerador de onda arbitrária (AWG), um instrumento de laboratório comum – isto é retratado na imagem da esquerda. Observe que a taxa de amostragem limitada descreve o sinal. O sinal produzido pelo AWG pode ser misturado com um gerador de ondas senoidais contínuas. A frequência de sua saída é controlada por instruções para o gerador de ondas senoidais; veja a imagem do meio. Finalmente, o sinal enviado para o qubit é demonstrado pelo lado direito da imagem, abaixo.

Nota: O hardware pode ser implementado de outras formas, mas se mantivermos as instruções separadas, evitamos perder informação explícita, como o valor da frequência de modulação.

Há muitos métodos disponíveis para a construção de pulsos. A nossa library, dentro do Qiskit Pulse, contém métodos úteis para a construção de Pulses. Tomemos, por exemplo, um pulso Gaussiano simples – um pulso com seu envelope, descrito por uma função gaussiana amostrada. Escolhemos, arbitrariamente, uma amplitude de 1, um desvio-padrão \(\sigma\) de 10, e 128 pontos de amostragem.

Nota: A amplitude máxima permitida é 1.0. A maioria dos sistemas também têm restrições adicionais no número mínimo e máximo de amostras permitidas em um pulso. Estas restrições adicionais, se disponíveis, seriam fornecidas, através da BackendConfiguration, que é descrita aqui.

[3]:

from qiskit.pulse import library

amp = 1
sigma = 10
num_samples = 128

Pulsos paramétricos¶

Vamos construir o nosso pulso Gaussiano usando o pulso paramétrico Gaussian. Um pulso paramétrico envia o nome da função e seus parâmetros para o backend, em vez de cada amostra individual. Usar pulsos paramétricos faz com que as tarefas (jobs), que você envia para o backend, sejam muito menores. Os serviços da IBM Quantum limitam o tamanho máximo da tarefa que eles aceitam, portanto pulsos paramétricos podem permitir que você execute programas maiores.

Outros pulsos paramétricos na library incluem GaussianSquare, Drag, e ConstantPulse.

Nota: O serviço é responsável por decidir, exatamente, como amostrar os pulsos paramétricos. É possível desenhar pulsos paramétricos, mas as amostras exibidas não são, necessariamente, as mesmas que são executadas no backend.

[4]:

from qiskit.pulse import Gaussian

pulse = Gaussian(num_samples, amp, sigma, name="Parametric Gaus")
pulse.draw()

[4]:

Formas de onda de pulso descritas por amostras

Também é possível especificar a forma de onda, como uma matriz de amostras. Passamos as amostras para um Waveform.

[5]:

import numpy as np

times = np.arange(num_samples)
gaussian_samples = np.exp(-1/2 *((times - num_samples / 2) ** 2 / sigma**2))

pulse = library.Waveform(gaussian_samples, name="WF Gaus")
pulse.draw()

[5]:

Funções da biblioteca Pulse

A nossa própria biblioteca de pulsos tem métodos de amostragem para criar uma Waveform de funções comuns.

[6]:

pulse = library.gaussian(duration=num_samples, amp=amp, sigma=sigma, name="Lib Gaus")
pulse.draw()

[6]:

Bibliotecas externas¶

Alternativamente, você pode fazer uso de uma biblioteca externa.

[7]:

from scipy import signal

sampled_gaussian_envelope = signal.gaussian(num_samples, sigma)
pulse = library.Waveform(sampled_gaussian_envelope, name="Scipy Gaus")
pulse.draw()

[7]:

Independentemente, de qual método você use para especificar o seu pulse, Play é instanciada da mesma forma:

[8]:

from qiskit.pulse import Play


play_gaus = Play(pulse, channel)

A instrução Play obtém sua duração, a partir do seu Pulse: a duração de um pulso parametrizado é um argumento explícito, e a duração de uma Waveform é o número de amostras de entrada.

## SetFrequency

Como explicado anteriormente, o envelope da forma de onda do pulso de saída também é modulado por uma frequência e fase. Cada canal possui uma frequência padrão listada no ``backend.defaults()` <5_gathering_system_information.ipynb#Defaults>`__.

A frequência de um canal pode ser atualizada, a qualquer momento, dentro de uma Schedule pela instrução SetFrequency. Ela recebe um float frequency e um channel PulseChannel como entrada. Todos os pulsos em um canal, depois de uma instrução SetFrequency, serão modulados pela frequência dada, até que outra instrução SetFrequency seja encontrada ou, até que o programa termine.

A instrução tem uma duração implícita de 0.

Nota: As frequências, que podem ser solicitadas, são limitadas pela largura de banda total e pela largura de banda instantânea de cada canal de hardware. No futuro, estas serão reportadas pelo backend.

[9]:

from qiskit.pulse import SetFrequency

set_freq = SetFrequency(4.5e9, channel)

## ShiftPhase

A instrução ShiftPhase aumentará a fase da modulação de frequência por phase. Como na SetFrequency, este deslocamento de fase afetará todas as instruções seguintes no mesmo canal, até que o programa termine. Para desfazer o efeito de uma ShiftPhase, a phase negativa pode ser passada para uma nova instrução.

Tal como a SetFrequency, a instrução tem uma duração implícita de 0.

[10]:

from qiskit.pulse import ShiftPhase

phase_pi = ShiftPhase(np.pi, channel)

Acquire¶

A instrução Acquire aciona a aquisição de dados para leitura. Ela recebe uma duração, um AcquireChannel, que mapeia para o qubit sendo medido e um MemorySlot ou um RegisterSlot. O MemorySlot é memória clássica, onde o resultado da leitura será armazenado. O RegisterSlot mapeia para um registrador na electrónica de controle, que armazena o resultado de leitura para feedback rápido.

Instruções Acquire também podem receber Discriminators e Kernels personalizados como argumentos de palavra-chave. Leia mais sobre a criação de medições aqui.

[11]:

from qiskit.pulse import Acquire, AcquireChannel, MemorySlot

acquire = Acquire(1200, AcquireChannel(0), MemorySlot(0))

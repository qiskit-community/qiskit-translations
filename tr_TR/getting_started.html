Kurulum¶

Qiskit, Python 3.6 veya daha üst sürümleri desteklemektedir. Fakat, hem Python hem de Qiskit devamlı evrilen ekosistemler olduğu için herhangi birinin yeni sürümünün çıkması durumunda uyumluluk problemleri ortaya çıkabilir.

Bilimsel programlama için çok platformlu Python dağıtımı olan Anaconda yı kurmanızı tavsiye ederiz. Anaconda bünyesindeki Jupyter, Qiskit ile etkileşime girmek için tavsiye edilir.

Qiskit, aşağıdaki 64 bitlik sistemler üzerinde test edilmiş ve desteklenmektedir:

  • Ubuntu 16.04 veya üzeri

  • macOS 10.12.6 veya üzeri

  • Windows 7 veya üzeri

Qiskit’i diğer uygulamalardan net bir şekilde ayırabilmek ve deneyiminizi geliştirmek için Python sanal ortamınlarını kullanmanızı tavsiye ederiz.

Ortamları kullanmanın en basit yolu Anaconda’nın içindeki conda komutunu kullanmaktır. Bir Conda ortamı Python’ın özel bir sürümünü belirtmenize ve kütüphaneler kurmanıza imkan sağlar. Çalışmak istediğiniz klasör içinde bir terminal penceresi açın.

Anaconda ile kurulan Anaconda komut istemini kullanmanız tercih edilir. Tek yapmanız gereken Anaconda’da sanal bir ortam oluşturmak ve ortamı etkinleştirmek. Bu komutlar, Windows veya Linux makinesinden bağımsız olarak Anaconda komut isteminde çalıştırılabilir.

Yalnızca üzerinde Python kurulu olan minimal bir ortam oluşturun.

conda create -n ENV_NAME python=3

Yeni ortamınızı etkinleştirin.

conda activate ENV_NAME

Daha sonra, Qiskit paketini kurun.

pip install qiskit

Paketler doğru kurulmuşsa sanal ortamınızdaki aktif paketleri görmek için conda list çalıştırabilirsiniz.

Görselleştirme işlevini veya Jupyter not defterlerini kullanmayı planlıyorsanız, ek görselleştirme desteği ile Qiskit’i kurmanız önerilir:

pip install qiskit[visualization]

Bir zsh kullanıcısıysanız (bu, macOS’un daha yeni sürümlerinde varsayılan kabuktur), tırnak işaretleri arasına qiskit[visualization] eklemeniz gerekeceğini belirtmekte fayda var:

pip install 'qiskit[visualization]'

Aşağıdaki bulut sağlayıcılarının ortamlarında, önceden kurulmuş Qiskit bulunur:

IBM Quantum Lab

Build quantum applications and experiments with Qiskit in a cloud programming environment.

Strangeworks

A platform that enables users and organizations to easily apply quantum computing to their most pressing problems and research.

Öğelerin kaynaktan kurulması, Python Package Index (PyPI) havuzundaki sürümü kullanmak yerine en son güncellenen Qiskit sürümüne erişmenizi sağlamaktadır. Bu da size Qiskit kodunun en son sürümünü daha verimli bir şekilde inceleme ve genişletme olanağı sağlar.

Ögeleri ve bileşenleri kaynaktan yüklerken, varsayılan olarak development sürümü (master git branch ına karşılık gelen), stable sürümün (yayınlanan pip paketleri ile aynı kod tabanı sürümü içeren) aksine kullanılacaktır. Bir ögenin veya bileşenin development sürümleri genellikle yeni özellikler ve değişiklikler içerdiğinden geri kalan ögelerin development sürümünün de kullanılmasını gerektirirler.

Kurmadan önce kaynaktan derlemek için Terra ve Aer paketlerinin ikisi de bir derleyici gerektirir. Ignis, Aqua ve IBM Quantum Provider backendi bir derleyici gerektirmez.

Ögelerin kaynaktan kurulması, eğer pip versiyonu kaynak sürümlerin gerisindeyse, istenenden daha düşük olabilecek öge sürümlerinin kurulmasını önlemek için aşağıdaki kurulum sırasını gerektirir:

  1. qiskit-terra

  2. qiskit-aer

  3. qiskit-ignis

  4. qiskit-aqua

  5. qiskit-ibmq-provider (IBM Quantum aygıtlarına ya da çevrimiçi simülatöre bağlanmak istiyorsanız)

Aynı anda birden çok bileşen ve öğeyle çalışmak istiyorsanız, her bir öğe için aşağıdaki adımları kullanın.

Not

Python’da namespace paketlemesinin kullanılması nedeniyle, paketlerin nasıl kurulduğuna dikkat edilmesi gerekmektedir. Kaynaktan herhangi bir öge yüklemeyi planlıyorsanız, qiskit meta paketini kullanmayın. Ayrıca, bu kılavuzu takip edin ve geliştirme için ayrı bir sanal ortam kullanın. Mevcut bir kurulumu geliştirmenizle karıştırmayı seçerseniz, birlikte çalışan kurulum yöntemi kombinasyonları için https://github.com/pypa/sample-namespace-packages/blob/master/table.md adresine bakınız.

Set up the Virtual Development Environment

conda create -y -n QiskitDevenv python=3
conda activate QiskitDevenv

Installing Terra from Source

Kaynaktan yükleme, sisteminizde C ++ 11’i destekleyen bir C ++ derleyicisine sahip olmanızı gerektirir.

Çoğu Linux platformunda, gerekli GCC derleyicisi zaten kuruludur.

MacOS kullanıyorsanız, XCode kurarak Clang derleyicisini kurabilirsiniz. Bir uçbirim penceresi açarak XCode ve Clang in kurulu olup olmadığını denetlemek için aşağıdakini giriniz.

clang --version

Aşağıdaki komutu kullanarak XCode ve Clang yazılımlarını yükleyin.

xcode-select --install

Windows’ta, Build Tools for Visual Studio 2019’dan Visual C++ derleyicisini indirmek en kolay yöntemdir. Bunun yerine, C++ derleyicisini yüklemek için gereken seçeneklerini işaretlediğinizden emin olarak Visual Studio sürüm 2015 veya 2017’yi yükleyebilirsiniz.

Derleyiciler kurulduktan sonra, Qiskit Terra’yı kurmaya hazırsınız.

  1. Terra deposunu klonlayın.

    git clone https://github.com/Qiskit/qiskit-terra.git

  2. Depoyu klonlamak qiskit-terra adında yerel bir klasör oluşturur.

    cd qiskit-terra

  3. Python gereksinim kütüphanelerini qiskit-terra dizininizden kurun.

    pip install cython

  4. Test veya linting (tiftikleme) kontrolleri yapmak istiyorsanız, geliştirici gereksinimlerini yükleyin.

    pip install -r requirements-dev.txt

  5. qiskit-terra’yı yükleyin.

    pip install .

Düzenlenebilir modda yüklemek istiyorsanız, yani projedeki kod değişikliklerinin yeniden yüklenmesi gerekmiyorsa, bunu şu şekilde yapabilirsiniz:

pip install -e .

Terra kurulumundan sonra kod örneklerini çalıştırabilirsiniz. Örneği aşağıdaki komutla çalıştırabilirsiniz.

python examples/python/using_qiskit_terra_level_0.py

Not

Başka bir bileşen kurmak istemiyorsanız eğer, qiskit-terra, hem qiskit-aer hem de qiskit-ibmq-provider yüklü olmadığı konusunda bir RunTimeWarning uyarısı verecektir. Bu, kullanıcılar genellikle ek öğeleri kullanmayı düşündükleri ancak yüklü olmadıklarının farkında olmadıkları veya Aer veya IBM Kuantum Sağlayıcısı’nın kurulumunun bir nedenden dolayı başarısız olduğu için yapılır. Bu uyarıları gizlemek istiyorsanız, kodunuza şunu ekleyin:

import warnings
warnings.filterwarnings('ignore', category=RuntimeWarning,
                        module='qiskit')

bunu kodunuza herhangi bir qiskit aktarmadan önce yapın. Bu, eksik qiskit-aer ve qiskit-ibmq sağlayıcısı hakkındaki uyarıyı bastırır, ancak qiskit veya diğer paketlerden gelen diğer uyarıları göstermeye devam eder.

Installing Aer from Source

  1. Aer deposunu klonla.

    git clone https://github.com/Qiskit/qiskit-aer

  2. Oluşturma gereksinimlerini kur.

    pip install cmake scikit-build cython

Bundan sonra, Aer’i yükleme adımları kullandığınız işletim sistemine bağlıdır. Aer, Python arayüzüyle derlenmiş bir C++ programı olduğundan, işletim sistemine bağlı olarak evrensel olarak yüklenemeyen Aer ikili dosyasını oluşturmak için Python dışında bağımlılıklar mevcuttur.

Linux

  1. Derleyici gereksinimlerini yükleyin.

    Aer’i kurmak bir C++ derleyicisi ve geliştirme başlıklarını gerektirir.

    Fedora veya ona eşdeğer bir Linux dağıtımı kullanıyorsanız aşağıdakileri kullanarak yükleyin:

    dnf install @development-tools

    Ubuntu/Debian için şunu kullanarak yükleyin:

    apt-get install build-essential

  2. OpenBLAS geliştirme üstbilgilerini kurun.

    Fedora veya ona eşdeğer bir Linux dağıtımı kullanıyorsanız aşağıdakileri kullanarak yükleyin:

    dnf install openblas-devel

    Ubuntu/Debian için şunu kullanarak yükleyin:

    apt-get install libopenblas-dev
macOS

  1. Bağımlılıkları kurun.

    MacOS’ta Clang derleyicisini kullanmak için OpenMP ‘i destekleyen fazladan bir kitaplık yüklemeniz gerekir. Bunu ve diğer bağımlılıkları yüklemek için brew kullanabilirsiniz.

    brew install libomp

  2. Sonra bir BLAS uygulaması yükleyin; OpenBLAS <https://www.openblas.net/> __ varsayılan seçimdir.

    brew install openblas

    Daha sonra, Xcode Command Line Tools programını yükleyin.

    xcode-select --install
Windows

Windows’ta tüm bağımlılıkları yüklemek için Anaconda3 veya Miniconda3 kullanmanız gerekir.

  1. Derleyici gereksinimlerini yükleyin.

    conda install --update-deps vs2017_win-64 vs2017_win-32 msvc_runtime

Qiskit Aer kuantum devreleri için yüksek performanslı bir simülatör sistemidir ve farklı simülasyon hedeflerini başarıyla gerçekleştirmek için birçok arkauç <apidoc/providers/aer/aer_backends.html> sağlar.

conda install --update-deps -c conda-forge -y openblas cmake

  1. Doğrudan qiskit-aer oluştur ve kur

    Sisteminizde pip < 19.0.0 kuruluysa ve ortamınız özel bir oluşturma gerektirmiyorsa, şunu çalıştırın:

    cd qiskit-aer
pip install .

    Bu, ikili dosyaları oluşturur ve Aer ‘i kurar.

    Alternatif olarak, daha yeni bir pip’iniz varsa veya bazı özel gereksinimleriniz varsa, manuel olarak bir Python çarkı oluşturabilirsiniz.

    cd qiskit-aer
python ./setup.py bdist_wheel

    Çark oluşturma sırasında özel bir seçenek ayarlamanız gerekirse, buraya bakınız Çark oluşturma sırasında özel seçenekler.

    Python çarkını oluşturduktan sonra, Aer deposundaki dist/ dizininde saklanacaktır. Tam sürümü şuna bağlıdır,

    cd dist
pip install qiskit_aer-*.whl

    Çıktı çarkı dosyasının tam dosya adı, geliştirilmekte olan Aer’in geçerli sürümüne bağlıdır.

Custom options

Aer derleme sistemi, derlemeyi Python arayüzüyle oluştururken çalıştırmak için scikit-build kullanılmaktadır. CMake çağırma ve yerel sisteminiz için ikili dosyaları derlemede `setuptools <https://setuptools.readthedocs.io/en/latest/>__ için bir arayüz görevi görür.

İkili dosyaları derlemenin karmaşıklığı nedeniyle, oluşturma işleminin belirli bir bölümüne seçenekleri geçirmeniz gerekebilir. Değişkenleri geçirmenin yolu:

python setup.py bdist_wheel [skbuild_opts] [-- [cmake_opts] [-- build_tool_opts]]

köşeli parantez [] içindeki öğeler isteğe bağlıdır ve skbuild_opts, cmake_opts, build_tool_opts, seçtiğiniz bayraklarla değiştirilir. *CMake * seçeneklerinin bir listesini burada bulabilirsiniz: https://cmake.org/cmake/help/v3.6/manual/cmake.1.html#options. Örneğin, şöyle bir şey çalıştırabilirsiniz:

python setup.py bdist_wheel -- -- -j8

Bu, -j8 bayrağını alttaki oluşturma sistemine geçirir (bu durumda Automake), 8 işlem kullanarak paralel şekilde oluşturmak istediğinizi söyler.

Örneğin, linux üzerindeki bu bayraklar için yaygın bir kullanım durumu, kullanılacak C++ derleyicisinin belirli bir sürümünü belirtmektir (normalde varsayılan çok eski ise):

python setup.py bdist_wheel -- -DCMAKE_CXX_COMPILER=g++-7

bu da CMake’e Aer derlenirken varsayılan g++ yerine g++-7 komutunu kullanmasını söyleyecektir.

Bunun için bir diğer yaygın kullanım durumu da, ortamınıza bağlı olarak platform adınızı belirtmeniz ve statik bağlantıyı kapatmanız gerekebileceğidir.

python setup.py bdist_wheel --plat-name macosx-10.9-x86_64 \
-- -DSTATIC_LINKING=False -- -j8

Burada --plat-name, kurulum araçları aracı için paket meta verilerinde kullanılacak platform adını belirtirken kullanılan bir bayraktır, -DSTATIC_LINKING, CMake’i statik bağlantıyı devre dışı bırakmada kullanmak için bir bayrak ve -j8 Automake’in derlemede 8 işlem kullanması için kullanılan bir bayraktır.

Installing Ignis from Source

  1. Ignis deposunu klonla.

    git clone https://github.com/Qiskit/qiskit-ignis.git

  2. Deponun klonlanması, qiskit-ignis adlı yerel bir dizin oluşturur.

    cd qiskit-ignis

  3. Test veya programlama hataları için kontroller yürütmek istiyorsanız, geliştirici gereksinimlerini yükleyin. Kaynaktan kurulum yaparken qiskit-ignis paketini kurmak veya kullanmak için bu gerekli değildir.

    pip install -r requirements-dev.txt

  4. Ignis’i yükle.

    pip install .

Düzenlenebilir modda kurulum gerçekleştirmek istiyorsanız, projedeki kod değişikliklerinin yeniden yüklenmesi gerekmez:

pip install -e .

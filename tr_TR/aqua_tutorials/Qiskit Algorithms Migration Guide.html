Qiskit Algoritmaları Geçiş Kılavuzu¶

Uygulamaların yeniden yapılandırılması

Qiskit 0.25.0 sürümü, uygulamaların ve algoritmaların yeniden yapılandırılmasını içerir. Daha önce Qiskit Aqua olarak adlandırılan, Qiskit’in tek uygulamalar ve algoritmalar modülü, şimdi Optimizasyon, Finans, Makine Öğrenimi ve Doğa (Fizik ve Kimya dahil) için özel uygulama modüllerine bölünmüştür. Çekirdek algoritmalar ve opflow operatörü işlevselliği Qiskit Terra’ya taşındı.

Algoritma arayüzleri

Yeniden yapılandırmaya ek olarak, tüm algoritmalar yeni bir birleşik paradigmayı takip eder: algoritmalar çözdüğü problemlere göre sınıflandırılır ve bir uygulama sınıfı içinde aynı problemi çözmek için algoritmalar birbirinin yerine kullanılabilir. Bu, öncekinden farklı olarak, algoritma örneklerinin çözdüğü problemden ayrıldığı anlamına gelir. Bunu bir akış şemasında özetleyebiliriz:

Örneğin, değişken kuantum özçözücü, “VQE”, bir operatörün minimum özdeğerini hesapladığı için bir “Minimum Özçözücü”dür. Buradaki problem, özdeğerini aradığımız operatör ile belirtilirken, varyasyonel yaklaşım devresi ve klasik optimizer gibi özellikler algoritmanın özellikleridir. Bu, “VQE”nin aşağıdaki yapıya sahip olduğu anlamına gelir

vqe = VQE(ansatz, optimizer)
result = vqe.compute_minimum_eigenvalue(operator)

Operatörünüzün özdeğerlerini hesaplamak için “MinimumEigensolver” arayüzünü uygulayan diğer herhangi bir algoritma ile “VQE”yi değiştirebiliriz, örneğin

numpy_based = NumPyMinimumEigensolver()
classical_reference = numpy_based.compute_minimum_eigenvalue(operator)

Bu, farklı algoritmalar arasında kolayca geçiş yapmanıza, klasik referansları kontrol etmenize ve kendi uygulamanızı sağlamanıza izin verir \(-\) sadece mevcut arayüzü uygulamanız yeterlidir.

Bu not defteri Qiskit Aqua kullanarak mevcut kodunuzu yeni yapıya dönüştürmeyi kolaylaştırmak için geçiş kılavuzu olarak hizmet eder.

Bu not defteri için kullanımdan kaldırma uyarısını devre dışı bırakıyoruz, böylece ``qiskit.aqua``dan bir nesne başlattığımızda hiçbir şey görmeyeceksiniz. Ancak, paketin tamamının kullanımdan kaldırıldığını ve aşağıdaki gibi bir uyarı vereceğini unutmayın:

[1]:

from qiskit.aqua.components.optimizers import COBYLA

optimizer = COBYLA()

Taşınan Opflow operatörleri

için

Öncesinde

from qiskit.aqua.operators import X, I, Y

** Yeni:**

[6]:

Ek özellikler:

qiskit.opflow ile, Pauli dizilerinin toplamlarının yeni ve daha verimli bir temsiliyle tanışıyoruz; bu, çok büyük Paulis toplamları üzerindeki hesaplamaları önemli ölçüde hızlandırabilir. Pauli karakter dizeleri toplanırsa, bu verimli gösterim otomatik olarak kullanılır:

[7]:

op = (X ^ X ^ Y ^ Y) + (X ^ 4) + (Y ^ 4) + (I ^ X ^ I ^ I)
type(op)

Grover¶

Özet¶

Önceki yapı

grover = Grover(oracle_settings, algorithm_settings)
result = grover.run()

problem/kara kutu ayarlarını ve algoritma ayarlarını bölmek için değiştirildi,

grover = Grover(algorithm_settings)
problem = AmplificationProblem(oracle_settings)
result = grover.amplify(problem)

Geçiş kılavuzu¶

Devreler olarak sağlanan kara kutular ve iyi durumları belirlemek için bir is_good_state işlevi

[10]:

from qiskit.circuit import QuantumCircuit

oracle = QuantumCircuit(2)
oracle.cz(0, 1)

def is_good_state(bitstr):
    return sum(map(int, bitstr)) == 2

Öncesinde

[11]:

from qiskit.aqua.algorithms import Grover

grover = Grover(oracle, is_good_state, quantum_instance=aqua_qinstance)
result = grover.run()
print('Top measurement:', result.top_measurement)

Top measurement: 11

** Yeni:**

[12]:

from qiskit.algorithms import Grover, AmplificationProblem

problem = AmplificationProblem(oracle=oracle, is_good_state=is_good_state)
grover = Grover(quantum_instance=qinstance)
result = grover.amplify(problem)
print('Top measurement:', result.top_measurement)

Top measurement: 11

“QuantumCircuit” sınıfını temel ilkel olarak kullanmak için tüm algoritmaları düzene soktuğumuzdan, “qiskit.aqua.compontents.Oracle” sınıfını kullanarak kara kutu tanımlamak artık tercih edilmemektedir. Örneğin LogicalExpressionOracle kullanmak yerine artık devre kütüphanesinden PhaseOracle devresini kullanabilirsiniz.

Öncesinde

[13]:

from qiskit.aqua.components.oracles import LogicalExpressionOracle
from qiskit.aqua.algorithms import Grover

oracle = LogicalExpressionOracle('x & ~y')
grover = Grover(oracle, quantum_instance=aqua_qinstance)
result = grover.run()
print('Top measurement:', result.top_measurement)

Top measurement: 01

** Yeni:**

[14]:

from qiskit.circuit.library import PhaseOracle
from qiskit.algorithms import Grover, AmplificationProblem

oracle = PhaseOracle('x & ~y')
problem = AmplificationProblem(oracle=oracle, is_good_state=oracle.evaluate_bitstring)
grover = Grover(quantum_instance=qinstance)
result = grover.amplify(problem)
print('Top measurement:', result.top_measurement)

Top measurement: 11

qiskit.aqua.components.oracles.TruthTableOracle henüz taşınmadı, ancak davranış, qiskit.circuit.classicalfunction modülü ile kolayca gerçekleştirilebilir, Grover algoritması hakkındaki öğretici derslere bakın.

Daha fazla örnek¶

To construct the circuit we can call construct_circuit and pass the problem instance we are interested in:

[15]:

power = 2
grover.construct_circuit(problem, power).draw('mpl', style='iqx')

Genlik tahmini¶

Özet¶

Tüm genlik tahmin algoritmaları için * AmplitudeEstimation * * IterativeAmplitudeEstimation * MaximumLikelihoodAmplitudeEstimation ve * FasterAmplitudeEstimation

arayüz değişti

qae = AmplitudeEstimation(algorithm_settings, estimation_settings)
result = qae.run()

problem/kara kutu ayarlarını ve algoritma ayarlarını bölmek için

qae = AmplitudeEstimation(algorithm_settings)
problem = EstimationProblem(oracle_settings)
result = qae.amplify(problem)

Geçiş kılavuzu¶

Burada, tek kübitimizde bir \(|1\rangle\) ölçme olasılığını tahmin etmek istiyoruz. Devre olarak durum hazırlığı sağlanmışsa,

[16]:

import numpy as np

probability = 0.25
rotation_angle = 2 * np.arcsin(np.sqrt(probability))

state_preparation = QuantumCircuit(1)
state_preparation.ry(rotation_angle, 0)

objective_qubits = [0]  # the good states are identified by qubit 0 being in state |1>

print('Target probability:', probability)

state_preparation.draw(output='mpl', style='iqx')

Target probability: 0.25

[16]:

Öncesinde

[17]:

from qiskit.aqua.algorithms import AmplitudeEstimation

# instantiate the algorithm and passing the problem instance
ae = AmplitudeEstimation(3, state_preparation, quantum_instance=aqua_qinstance)

# run the algorithm
result = ae.run()

# print the results
print('Grid-based estimate:', result.estimation)
print('Improved continuous estimate:', result.mle)

Grid-based estimate: 0.1464466
Improved continuous estimate: 0.2499999956539467

Şimdi:

[18]:

from qiskit.algorithms import AmplitudeEstimation, EstimationProblem

problem = EstimationProblem(state_preparation=state_preparation, objective_qubits=objective_qubits)
ae = AmplitudeEstimation(num_eval_qubits=3, quantum_instance=qinstance)

result = ae.estimate(problem)
print('Grid-based estimate:', result.estimation)
print('Improved continuous estimate:', result.mle)

Grid-based estimate: 0.1464466
Improved continuous estimate: 0.2499999956513743

Başka bir endeks belirtilmemişse, eski sınıfın “durum_hazırlığı”ndaki son kübiti varsayılan olarak nesnel kübit olarak kullandığını unutmayın. Bu varsayılan, şeffaflığı artırmak ve örtük varsayımları kaldırmak için artık mevcut değil.

Daha fazla örnek¶

Genlik tahmini için devreyi kurmak için şunları yapabiliriz

[19]:

ae.construct_circuit(estimation_problem=problem).draw('mpl', style='iqx')

Minimum Öz değerlikler¶

Özet¶

  • Arayüz çoğunlukla aynı kaldı, ancak daha önce operatörün başlatıcıya iletilmesinin mümkün olduğu yerlerde.

  • Operatörler artık “qiskit.aqua.operators” yerine “qiskit.opflow”dan operatörlerle oluşturulmalıdır.

  • VQE argümanı var_form ansatz olarak yeniden adlandırıldı.

Geçiş kılavuzu¶

Minimum özdeğerini bulmak istediğimizi varsayın

\[H = Z \otimes I.\]

NumPy tabanlı öz çözücü¶

Öncesinde

Daha önce operatörleri ``qiskit.aqua.operators``dan içe aktarmıştık:

[22]:

from qiskit.aqua.operators import Z, I

observable = Z ^ I

ve daha sonra kullanılarak, minimum özdeğer için çözüldü

[23]:

from qiskit.aqua.algorithms import NumPyMinimumEigensolver

mes = NumPyMinimumEigensolver()
result = mes.compute_minimum_eigenvalue(observable)
print(result.eigenvalue)

(-1+0j)

Önceden, problem-algoritma ayrımı nedeniyle artık izin verilmeyen başlatıcıda gözlemlenebiliri iletmek mümkündü.

[24]:

mes = NumPyMinimumEigensolver(observable)
result = mes.compute_minimum_eigenvalue()
print(result.eigenvalue)

(-1+0j)

Şimdi:

Şimdi qiskit.opflow dosyasından içe aktarmamız gerekiyor, ancak diğer sözdizimi tamamen aynı kalır:

[25]:

from qiskit.opflow import Z, I

observable = Z ^ I

[26]:

from qiskit.algorithms import NumPyMinimumEigensolver

mes = NumPyMinimumEigensolver()

result = mes.compute_minimum_eigenvalue(observable)
print(result.eigenvalue)

(-1+0j)

HHL¶

Özet¶

HHL, sezgisel bir arayüze izin vermek ve sonucun verimli, devre tabanlı bir temsilini döndürmek için tamamen yeniden düzenlendi.

Geçiş kılavuzu¶

Aşağıdaki lineer sistemi çözmek istediğimizi varsayalım

\[\begin{split}\begin{pmatrix} 1 & -1/3 \\ -1/3 & 1 \\ \end{pmatrix} \vec x = \begin{pmatrix} 1 \\ 0 \end{pmatrix}\end{split}\]
[42]:

import numpy as np

matrix = np.array([[1, -1/3], [-1/3, 1]])
vector = np.array([1, 0])

Öncesinde

[43]:

from qiskit.circuit.library import QFT
from qiskit.aqua.algorithms import HHL
from qiskit.aqua.components.eigs import EigsQPE
from qiskit.aqua.components.reciprocals import LookupRotation
from qiskit.aqua.components.initial_states import Custom
from qiskit.aqua.operators import MatrixOperator

def create_eigs(matrix, num_auxiliary, num_time_slices, negative_evals):
    ne_qfts = [None, None]
    if negative_evals:
        num_auxiliary += 1
        ne_qfts = [QFT(num_auxiliary - 1), QFT(num_auxiliary - 1).inverse()]
    return EigsQPE(MatrixOperator(matrix=matrix),
                   QFT(num_auxiliary).inverse(),
                   num_time_slices=num_time_slices,
                   num_ancillae=num_auxiliary,
                   expansion_mode='suzuki',
                   expansion_order=2,
                   evo_time=None,
                   negative_evals=negative_evals,
                   ne_qfts=ne_qfts)

orig_size = len(vector)

matrix, vector, truncate_powerdim, truncate_hermitian = HHL.matrix_resize(matrix, vector)

# Initialize eigenvalue finding module
eigs = create_eigs(matrix, 3, 50, False)
num_q, num_a = eigs.get_register_sizes()

# Initialize initial state module
init_state = Custom(num_q, state_vector=vector)

# Initialize reciprocal rotation module
reci = LookupRotation(negative_evals=eigs._negative_evals, evo_time=eigs._evo_time)
algo = HHL(matrix, vector, truncate_powerdim, truncate_hermitian, eigs,
           init_state, reci, num_q, num_a, orig_size)

result = algo.run(aqua_qinstance)
print(result.solution)

Faz tahmini¶

Özet¶

Faz tahmini tamamen yeniden düzenlendi ve bir Hamiltoniyenin özdeğerini hesaplamak için kullanılan sadece bir “qiskit.aqua.algorithms.QPE” sınıfı yerine, şimdi iki ayrı uygulamamız var: “HamiltonianPhaseEstimation” rolü üstlenen ders kitabı faz tahmini için eski “QPE” ve yeni “PhaseEstimation” algoritması.

Tekrar eden aşama tahmini, qiskit.aqua.algorithms.IQPE henüz yenilenmedi, ancak yakında olacak.

Geçiş kılavuzu¶

Haydı özdeğerini bulma problemini ele alalım

\[H = 0.5 X + Y + Z\]

giriş durumuyla \(|0\rangle\).

[47]:

state_in = np.array([1, 0])

Öncesinde

[48]:

from qiskit.circuit.library import QFT
from qiskit.aqua.algorithms import QPE
from qiskit.aqua.components.initial_states import Custom
from qiskit.aqua.operators import I, X, Y, Z

n_ancillae = 5
num_time_slices = 1

op = 0.5 * X + Y + Z

state_preparation = Custom(op.num_qubits, state_vector=state_in)
iqft = QFT(n_ancillae, do_swaps=False).inverse().reverse_bits()

qpe = QPE(op, state_preparation, iqft, num_time_slices, n_ancillae,
          expansion_mode='trotter',
          shallow_circuit_concat=True)


result = qpe.run(aqua_qinstance)

print(result.eigenvalue)

0.78125

** Yeni:**

[49]:

from qiskit import BasicAer
from qiskit.algorithms import HamiltonianPhaseEstimation

from qiskit.aqua.algorithms import EOH
from qiskit.aqua.operators import WeightedPauliOperator
from qiskit.aqua.components.initial_states import Custom
from qiskit.quantum_info import Pauli

hamiltonian = WeightedPauliOperator([[1, Pauli('XX')], [1, Pauli('ZZ')], [1j, Pauli('YY')]])

observable = WeightedPauliOperator([[1, Pauli('XI')]])

initial_state = Custom(2, 'uniform')

evo_time = 2
num_time_slices = 10

eoh = EOH(observable, initial_state, hamiltonian, evo_time=evo_time, num_time_slices=num_time_slices)
result = eoh.run(aqua_qinstance)
result['avg']

** Yeni:**

[52]:

import numpy as np
from qiskit.opflow import I, X, Y, Z, PauliTrotterEvolution, StateFn

hamiltonian = (X ^ X) + (Z ^ Z) + 1j * (Y ^ Y)

observable = X ^ I

initial_state = StateFn(np.ones(4) / 2)

evo_time = 2
num_time_slices = 10

# get the evolution operator
evolved_hamiltonian = (evo_time * hamiltonian).exp_i()

evo = PauliTrotterEvolution(reps=num_time_slices)
evo.convert(evolved_hamiltonian)

# get the evolved state
evolved_state = evolved_hamiltonian @ initial_state

# evaluate the target observable at the evolved state
print((~StateFn(observable) @ evolved_state).eval())

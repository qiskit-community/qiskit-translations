Qiskit স্থাপন (ইন্সটল) করুন¶

যা যা প্রয়োজন¶

Qiskit, পাইথন ৩. ৬ বা তার পরের সংস্করণগুলোতে চলে। তা সত্ত্বেও, যেহেতু Qiskit ও পাইথন উভয়ই আপডেট হয়ে চলেছে, তাই একটার নতুন সংস্করণ অন্যটাতে কিছু অসুবিধা করতে পারে।

Python হালনাগাদ করার পরে যদি আপনার Qiskit স্থাপন (ইন্সটল) বা ব্যবহার করতে অসুবিধা হয়, তাহলে Qiskit প্যাকেজ মেটাডাটা -এর Programming Language অংশে দেখুন Python-এর কোন কোন সংস্করণ Qiskit অনুমোদন করে।

আমরা অ্যানাকোন্ডা, স্থাপন (ইন্সটল) করাকে উৎসাহিত করি যা বৈজ্ঞানিক গণনাকর্মের জন্য বিভিন্ন প্ল্যাটফর্মে ব্যবহৃত Python এর একটি ডিস্ট্রিবিউশন। Qiskit ব্যবহারের জন্য অ্যানাকোন্ডার অন্তর্ভুক্ত জুপিটার প্রস্তাব করা হয়ে থাকে।

Qiskit নিম্নলিখিত ৬৪ বিট ব্যবস্থাগুলোতে পরীক্ষিত এবং অনুমোদিত:

  • উবুন্টু ১৬.০৪ বা তার পরবর্তী সংস্করণ

  • ম্যাকওএস ১০.১২.৬ বা তার পরবর্তী সংস্করণ

  • উইন্ডোজ ৭ বা তার পরবর্তী সংস্করণ

স্থাপন (ইন্সটল) করুন¶

উন্নত অভিজ্ঞতা অর্জনের লক্ষ্যে, Qiskit কে অন্যান্য কার্যক্রম থেকে বিচ্ছিন্ন করে রাখার জন্য আমরা Python এর কৃত্রিম কর্মপরিবেশ (ভার্চুয়াল এনভায়রনমেন্ট) ব্যবহারের পরামর্শ দেই।

কর্মপরিবেশগুলো (এনভায়রনমেন্ট) ব্যবহারের সবচেয়ে সহজ উপায় হচ্ছে Anaconda এর অন্তর্ভুক্ত conda নির্দেশনা ব্যবহার করা। কোন্ডার (conda) কর্মপরিবেশ (এনভায়রনমেন্ট) Python (পাইথন ) একটি নির্দিষ্ট সংস্করণ এবং এর অন্তর্ভুক্ত লাইব্রেরিগুলো ব্যবহারের অনুমতি দেয়। আপনি কাজ করতে চান এমন একটি ডিরেক্টরিতে টার্মিনাল উইন্ডো চালু করে সেখানে কাজ করতে পারেন।

অ্যানাকোন্ডার সাথে যে অ্যানাকোন্ডা প্রম্পট ইন্সটল হয় সেটি ব্যবহার করা শ্রেয়। আপনাকে যা করতে হবে তা হলো, অ্যানাকোন্ডায় একটি কৃত্রিম কর্মপরিবেশ (ভার্চুয়াল এনভায়রনমেন্ট) তৈরি এবং সক্রিয় করতে হবে। উইন্ডোজ বা লিনাক্স যেকোনো অপারেটিং ব্যবস্থায় এই নির্দেশনাসমূহ চালানো যাবে।

শুধুমাত্র Python স্থাপন (install) করে একটি ঊন (minimal) কর্মপরিবেশ (এনভায়রনমেন্ট) তৈরি করুন।

conda create -n name_of_my_env python=3

আপনার নতুন কর্মপরিবেশ (এনভায়রনমেন্ট) সক্রিয় করুন।

conda activate name_of_my_env

নোট

আপনি যদি কোন্ডা ৪.৬ এর পূর্ববর্তী সংস্করণ ব্যবহার করেন তাহলে লিনাক্স অথবা ম্যাকওএস ব্যবহারকারী হলে source activate name_of_my_env এবং উইন্ডোজ ব্যবহারকারী হলে activate name_of_my_env ব্যবহার করুন।

এখন টেরা, এয়ার, ইগ্নিস এবং অ্যাকোয়া সম্বলিত Qiskit প্যাকেজসমূহ স্থাপন করুন।

pip install qiskit

নোট

Qiskit ০.১৩.০ সংস্করণ দিয়ে শুরু করতে লিনাক্স ব্যবস্থায় পূর্ব-সম্পাদিত দ্বিমিক (বাইনারি) থেকে qiskit-এয়ার স্থাপন (ইন্সটল) করার জন্য পিপ ১৯ বা তার নতুনতর সংস্করণ প্রয়োজন। আপনার যদি পিপ ১৯ স্থাপন (ইন্সটল) না করা থাকে তবে আপনি pip install -U pip নির্দেশনাটি চালিয়ে এটি স্থাপন এবং নতুনতর সংস্করণে উন্নীত করতে পারবেন। পিপ ১৯ বা নতুনতর সংস্করণ ছাড়া এই নির্দেশনাটি qiskit-এয়ার sdist (উৎস সংস্করণ) থেকে স্থাপন করার চেষ্টা করবে যা এয়ারকে আড়ালে স্থানীয়ভাবে সম্পাদন (কম্পাইল) করে।

যদি প্যাকেজগুলো সঠিকভাবে স্থাপন করা হয়ে থাকে তবে কৃত্রিম কর্মপরিবেশে (ভার্চুয়াল এনভায়রনমেন্টে) সক্রিয় প্যাকেজগুলো দেখার জন্য আপনি conda list নির্দেশনাটি চালাতে পারেন।

আপনি যদি ভিজুয়ালাইজেশন (দৃশ্যায়ন) ফাংশনালিটি অথবা জুপিটার নোটবুক ব্যবহার করতে চান তাহলে ভিজুয়ালাইজেশনের (দৃশ্যায়ন) যথোপযুক্ত শর্তগুলি (রিকোয়ারমেন্টস) সমেত qiskit স্থাপন করুন।

pip install qiskit[visualization]

মনে করিয়ে দেয়া প্রয়োজন, আপনি যদি zsh (ম্যাকওএস এর নতুনতর সংস্করণগুলোতে এটি পূর্বনির্ধারিত (ডিফল্ট) শেল) ব্যবহারকারী হয়ে থাকেন তাহলে আপনাকে উদ্ধৃতির মধ্যে qiskit[visualization] লিখতে হবে:

pip install 'qiskit[visualization]'

নোট

স্থাপিত এবং অনুমোদিত Qiskit প্যাকেজসমূহ ব্যবহার উপযোগী হওয়ার পর, কাজ শুরু করার জন্য Python কর্মপরিবেশে (এনভায়রনমেন্ট) প্যাকেজগুলো আমদানি করুন।

import qiskit

নোট

আপনি যদি নিয়মিত Qiskit কমিউনিটির (সংগঠন) উন্নয়ন এবং সহযোগী অংশীদার হতে চান তাহলে এটি দেখুন - উৎস থেকে Qiskit প্যাকেজসমূহ গঠন।

আইবিএম কোয়ান্টাম ব্যবস্থায় প্রবেশ¶

Qiskit প্যাকেজ ও আইবিএম কোয়ান্টাম ব্যবস্থা দ্বারা উচ্চ ক্ষমতাসম্পন্ন ধ্রুপদী (ক্লাসিক্যাল) কম্পিউটারের মহড়া (সিমুলেটর) এবং সত্যিকারের কোয়ান্টাম কম্পিউটারের অভিজ্ঞতা পাওয়া সম্ভব। আইবিএম কোয়ান্টাম ব্যবস্থায় কর্ম (জব) সম্পাদনের জন্য Qiskit সেটআপ করতে নিম্নোক্ত ধাপগুলো অনুসরণ করুন।

নোট

আইবিএম কোয়ান্টাম অভিজ্ঞতার ক্রেডেনশিয়ালগুলো যদি আপনি পূর্বে স্থানীয়ভাবে সংরক্ষণ করে থাকেন, তবে Qiskit ০.১১ সংস্করণের ক্ষেত্রে, সেগুলো আপনার হালনাগাদ করা প্রয়োজন হতে পারে যদি আপনি আইবিএম কোয়ান্টাম অভিজ্ঞতা ভি২ (V2) ব্যবহার করতে চান। এজন্য আইবিএম কোয়ান্টাম অভিজ্ঞতার ক্রেডেনশিয়াল হালনাগাদকরণ দেখুন।

ব্যক্তিগত বিবরকটি (অ্যাকাউন্ট) কনফিগার করতে, এপিআই সংকেত সম্বলিত একটি স্থানীয় কনফিগারেশন ফাইল তৈরি করতে হবে।

১. `একটি আইবিএম কোয়ান্টাম অভিজ্ঞতা (এক্সপেরিয়েন্স) বিবরক (অ্যাকাউন্ট) তৈরি করুন <https://quantum-computing.ibm.com/login> `__ ।

২. ব্যক্তিগত বিবরকটি যাচাইয়ের জন্য আমার অ্যাকাউন্ট এ যান।

৩. কুপন (টোকেন) অনুলিপির জন্য Copy token চাপুন। পরবর্তীতে ব্যক্তিগত বিবরকটি (account) কনফিগারেশনের জন্য অনুলিপিকৃত এপিআই কুপনটি নিজের প্রিয় লিখন সম্পাদকে (টেক্সট এডিটর) সাময়িকভাবে পেস্ট করুন।

৪. এপিআই কুপনটি স্থানীয়ভাবে qiskitrc নামক কনফিগারেশন ফাইলে পরবর্তীতে ব্যবহারের জন্য সংরক্ষণ করতে নিচের নির্দেশনাগুলো কার্যকর করুন। আপনার টেক্সট এডিটরে সংরক্ষিত এপিআই টোকেনটি দিয়ে MY_API_TOKEN কে প্রতিস্থাপন করে দিন।

from qiskit import IBMQ
IBMQ.save_account('MY_API_TOKEN')

আরো তথ্য যেমন কীভাবে একাধিক আইবিএম কোয়ান্টাম বিবরক (অ্যাকাউন্ট) পরিচালনা করা যায়, জানার জন্য The IBM Quantum Account শিরোনামের এই নির্দেশিকা অনুসরণ করুন।

সংস্করণ যাচাইকরণ¶

যেহেতু, Qiskit প্যাকেজের মধ্যে বিভিন্নধরণের উপকরণ সন্নিবিষ্ট আছে, সেহেতু, Qiskit-এর সংস্করণ জানার জন্য শুধু qiskit.__version__ নির্দেশনাটি চালানো বিভ্রান্তির সৃষ্টি করতে পারে। কারণ, এটি শুধু qiskit-terra প্যাকেজের সংস্করণ জানায়। এর কারণ হচ্ছে Python-এ qiskit নেমস্পেস Qiskit প্যাকেজ থেকে আসেনা, বরং qiskit-terra প্যাকেজ থেকে আসে।

import qiskit
qiskit.__version__

'0.16.1'

আপনার তৈরিকৃত কর্মপরিবেশে সকল Qiskit উপাদানসমূহের সংস্করণ জানতে আপনি __qiskit_version__ এই নির্দেশনাটি ব্যবহার করতে পারেন। উদাহরণসরূপ, নিচের নির্দেশনাটি চালনা করলে এটি আপনাকে একটি অভিধান (ডিকশনারি) ফেরত দিবে যাতে আপনার স্থাপিত (ইন্সটলকৃত) সকল প্যাকেজের সংস্করণ উল্লেখ করা থাকবে।

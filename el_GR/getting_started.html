Εγκατάσταση¶

Το Qiskit υποστηρίζει Python 3.6 ή νεότερη έκδοση. Ωστόσο, τόσο η Python όσο και το Qiskit είναι εξελισσόμενα οικοσυστήματα, και μερικές φορές με τις νέες εκδόσεις στο ένα ή στο άλλο, μπορεί να υπάρχουν προβλήματα με τη συμβατότητα.

Συστήνουμε την εγκατάσταση του ` Anaconda <https://www.anaconda.com/download/>` __, μια πολλαπλή πλατφόρμα διανομής Python για την επιστημονική πληροφορική. Το Jupyter, που περιλαμβάνεται στο Anaconda, συνιστάται για την αλληλεπίδραση με το Qiskit.

Το Qiskit έχει δοκιμαστεί και υποστηρίζεται στα ακόλουθα 64-bit συστήματα:

  • Ubuntu 16.04 ή νεότερο

  • macOS 10.12.6 ή νεότερο

  • Windows 7 ή νεότερο

Συνιστούμε τη χρήση εικονικών περιβαλλόντων Python για να διαχωρίσετε το Qiskit από άλλες εφαρμογές και να βελτιώσετε την εμπειρία σας.

Ο απλούστερος τρόπος χρήσης των περιβαλλόντων είναι χρησιμοποιώντας την εντολή ``conda`, που περιλαμβάνεται στο Anaconda. Ένα περιβάλλον Conda σας επιτρέπει να καθορίσετε μια συγκεκριμένη έκδοση της Python και το σύνολο των βιβλιοθηκών. Ανοιξετε ένα παράθυρο τερματικού στον κατάλογο στον οποίο θέλετε να εργαστείτε.

Είναι προτιμότερο να χρησιμοποιείτε γραμμή εντολών Anaconda που έχει εγκατασταθεί με το Anaconda. Το μόνο που έχετε να κάνετε είναι να δημιουργήσετε ένα εικονικό περιβάλλον μέσα στο Anaconda και να ενεργοποιήσετε το περιβάλλον. Αυτές οι εντολές μπορούν να εκτελεστούν στην γραμμή εντολών Anaconda ανεξάρτητα από το μηχάνημα Windows ή Linux.

Δημιουργήστε ένα ελάχιστο περιβάλλον με εγκατεστημένη μόνο την Python.

conda create -n ENV_NAME python=3

Ενεργοποιήστε το νέο σας περιβάλλον.

conda activate ENV_NAME

Στη συνέχεια, εγκαταστήστε το πακέτο Qiskit.

pip install qiskit

Αν τα πακέτα έχουν εγκατασταθεί σωστά, μπορείτε να εκτελέσετε την εντολή ` ` conda list ` ` για να δείτε τα ενεργά πακέτα στο εικονικό περιβάλλον σας.

Αν σκοπεύετε να χρησιμοποιήσετε τη λειτουργικότητα virtualization ή τα Jupyter notebooks συνιστάται να εγκαταστήσετε το Qiskit με την επιπλέον υποστήριξη ``visualization`:

pip install qiskit[visualization]

Αξίζει να σημειωθεί ότι αν είστε χρήστης zsh (το οποίο είναι το προεπιλεγμένο κέλυφος σε νεότερες εκδόσεις του macOS), θα χρειαστεί να βάλεις το qiskit[visualization] σε εισαγωγικά:

pip install 'qiskit[visualization]'

Οι ακόλουθοι προμηθευτές cloud έχουν εγκατεστημένο το Qiskit στο περιβάλλον τους:

IBM Quantum Lab

Build quantum applications and experiments with Qiskit in a cloud programming environment.

Strangeworks

A platform that enables users and organizations to easily apply quantum computing to their most pressing problems and research.

Η εγκατάσταση των στοιχείων από την πηγή, σας επιτρέπει να αποκτήσετε πρόσβαση στην πιο πρόσφατη έκδοση του Qiskit αντί να χρησιμοποιήσετε την έκδοση στο αποθετήριο Python Package Index (PyPI). Αυτό θα σας δώσει τη δυνατότητα να επιθεωρήσετε και να επεκτείνετε την τελευταία έκδοση του κώδικα Qiskit πιο αποτελεσματικά.

Κατά την εγκατάσταση των στοιχείων και των συστατικών στοιχείων από την πηγή, από προεπιλογή θα χρησιμοποιηθεί η έκδοση development (η οποία αντιστοιχεί στο master git branch), σε αντίθεση με την έκδοση stable` (η οποία περιέχει την ίδια βάση κώδικα με τα δημοσιευμένα πακέτα pip). Από τις εκδόσεις development ενός στοιχείου ή συστατικού στοιχείου συνήθως περιλαμβάνουν νέα χαρακτηριστικά και αλλαγές, απαιτούν γενικά τη χρήση της έκδοσης development και των υπόλοιπων αντικειμένων.

Τα πακέτα Terra και Aer απαιτούν και τα δύο έναν μεταγλωττιστή να χτιστούν από την πηγή πριν να μπορέσετε να τα εγκαταστήσετε. Τα Ignis, Aqua, και το σύστημα υποστήριξης της IBM Quantum Provider δεν απαιτούν μεταγλωττιστή.

Η εγκατάσταση στοιχείων από την πηγή απαιτεί την ακόλουθη σειρά εγκατάστασης για να αποτραπεί η εγκατάσταση εκδόσεων στοιχείων που μπορεί να είναι μικρότερες από τις επιθυμητές, εάν η έκδοση `pip`είναι πίσω από τις εκδόσεις πηγής:

  1. :ref:»qiskit-terra <install-qiskit-terra>»

  2. qiskit-aer

  3. qiskit-ignis

  4. qiskit-aqua

  5. qiskit-ibmq-provider (αν θέλετε να συνδεθείτε με τις συσκευές IBM Quantum ή τον online προσομοιωτή)

Για να εργαστείτε με διάφορα συστατικά και στοιχεία ταυτόχρονα, χρησιμοποιήστε τα ακόλουθα βήματα για κάθε στοιχείο.

Σημείωση

Λόγω της χρήσης της συσκευασίας namespace στην Python, πρέπει να ληφθεί μέριμνα για τον τρόπο εγκατάστασης των πακέτων. Αν σχεδιάζετε να εγκαταστήσετε οποιοδήποτε στοιχείο από την πηγή, μην χρησιμοποιήσετε το μετα-πακέτο qiskit. Επίσης, ακολουθήστε αυτόν τον οδηγό και χρησιμοποιήστε ένα ξεχωριστό εικονικό περιβάλλον για ανάπτυξη. Αν επιλέξετε να αναμίξετε μια υπάρχουσα εγκατάσταση με την ανάπτυξή σας, ανατρέξτε στο https://github. om/pypa/sample-namespace-packages/blob/master/table.md για το σύνολο των συνδυασμών μεθόδων εγκατάστασης που λειτουργούν μαζί.

Set up the Virtual Development Environment

conda create -y -n QiskitDevenv python=3
conda activate QiskitDevenv

Installing Terra from Source

Η εγκατάσταση από την πηγή απαιτεί να έχετε ένα μεταγλωττιστή C++ στο σύστημά σας που υποστηρίζει C++11.

Στις περισσότερες πλατφόρμες Linux, είναι ήδη εγκατεστημένος ο απαραίτητος μεταγλωττιστής GCC.

Εάν χρησιμοποιείτε macOS, μπορείτε να εγκαταστήσετε το μεταγλωττιστή Clang εγκαθιστώντας το XCode. Ελέγξτε αν έχετε εγκαταστήσει το XCode και Clang ανοίγοντας ένα παράθυρο τερματικού και εισάγοντας τα παρακάτω.

clang --version

Εγκαταστήστε το XCode και Clang χρησιμοποιώντας την ακόλουθη εντολή.

xcode-select --install

Στα Windows, είναι ευκολότερο να εγκαταστήσετε τον μεταγλωττιστή Visual C++ από τα Εργαλεία Κατασκευής για το Visual Studio 2017. Αντ “αυτού μπορείτε να εγκαταστήσετε το Visual Studio έκδοση 2015 ή 2017, φροντίζοντας να επιλέξετε τις επιλογές για την εγκατάσταση του μεταγλωττιστή C++.

Μόλις εγκατασταθούν οι μεταγλωττιστές, είστε έτοιμοι να εγκαταστήσετε το Qiskit Terra.

  1. Κλωνοποιήστε το αποθετήριο Terra.

    git clone https://github.com/Qiskit/qiskit-terra.git

  2. Η κλωνοποίηση του αποθετηρίου δημιουργεί έναν τοπικό φάκελο που ονομάζεται qiskit-terra.

    cd qiskit-terra

  3. Εγκαταστήστε τις βιβλιοθήκες απαιτήσεων Python από τον κατάλογο qiskit-terra.

    pip install cython

  4. Αν θέλετε να εκτελέσετε δοκιμές ή ελέγχους σύνδεσης, εγκαταστήστε τις απαιτήσεις του προγραμματιστή.

    pip install -r requirements-dev.txt

  5. Εγκατάσταση ` ` qiskit-terra ` `.

    pip install .

Αν θέλετε να το εγκαταστήσετε σε επεξεργάσιμη λειτουργία, που σημαίνει ότι οι αλλαγές κώδικα στο έργο δεν απαιτούν την εφαρμογή επανεγκατάστασης, μπορείτε να το κάνετε με:

pip install -e .

Στη συνέχεια μπορείτε να εκτελέσετε τα παραδείγματα κώδικα μετά την εγκατάσταση του Terra. Μπορείτε να εκτελέσετε το παράδειγμα με την ακόλουθη εντολή.

python examples/python/using_qiskit_terra_level_0.py

Σημείωση

Αν δεν σκοπεύετε να εγκαταστήσετε άλλα στοιχεία, το qiskit-terra θα παρουσιάσει μια προειδοποίηση RuntimeWarning ότι και τα δύο qiskit-aer και qiskit-ibmq-provider δεν είναι εγκατεστημένοι. Αυτό γίνεται επειδή οι χρήστες συνήθως προτίθενται να χρησιμοποιήσουν τα πρόσθετα στοιχεία, αλλά δεν συνειδητοποιούν ότι δεν έχουν εγκατασταθεί, ή ότι η εγκατάσταση είτε της Aer είτε της IBM Quantum Provider απέτυχε για κάποιο λόγο. Αν θέλετε να καταργήσετε αυτές τις προειδοποιήσεις, προσθέστε:

import warnings
warnings.filterwarnings('ignore', category=RuntimeWarning,
                        module='qiskit')

πριν από οποιαδήποτε εισαγωγή qiskit στον κώδικά σας. Αυτό θα καταστείλει την προειδοποίηση για την απουσία qiskit-aer και qiskit-ibmq-provider, αλλά θα συνεχίσει να εμφανίζει οποιεσδήποτε άλλες προειδοποιήσεις από qiskit ή άλλα πακέτα.

Installing Aer from Source

  1. Κλωνοποίηση του αποθετηρίου Aer.

    git clone https://github.com/Qiskit/qiskit-aer

  2. Εγκατάσταση απαιτήσεων κατασκευής.

    pip install cmake scikit-build cython

Στη συνέχεια, τα βήματα για την εγκατάσταση Aer εξαρτώνται από το λειτουργικό σύστημα που χρησιμοποιείτε. Δεδομένου ότι Aer είναι ένα μεταγλωττισμένο πρόγραμμα C++ με διεπαφή Python, υπάρχουν μη-Python εξαρτήσεις για την κατασκευή του Aer binary, το οποίο δεν μπορεί να εγκατασταθεί καθολικά ανάλογα με το λειτουργικό σύστημα.

Linux

  1. Εγκατάσταση απαιτήσεων μεταγλωττιστή.

    Η κατασεκυή Aer απαιτεί έναν μεταγλωττιστή C++ και επικεφαλίδες ανάπτυξης.

    Εάν χρησιμοποιείτε το Fedora ή μια αντίστοιχη διανομή Linux, εγκαταστήστε χρησιμοποιώντας:

    dnf install @development-tools

    Για το Ubuntu/Debian εγκαταστήστε το χρησιμοποιώντας:

    apt-get install build-essential

  2. Εγκαταστήστε τις κεφαλίδες ανάπτυξης OpenBLAS.

    Εάν χρησιμοποιείτε το Fedora ή μια αντίστοιχη διανομή Linux, εγκαταστήστε χρησιμοποιώντας:

    dnf install openblas-devel

    Για το Ubuntu/Debian εγκαταστήστε το χρησιμοποιώντας:

    apt-get install libopenblas-dev
macOS

  1. Εγκαταστήστε τις εξαρτήσεις.

    Για να χρησιμοποιήσετε τον μεταγλωττιστή Clang στο macOS, πρέπει να εγκαταστήσετε μια επιπλέον βιβλιοθήκη για την υποστήριξη του OpenMP. Μπορείτε να χρησιμοποιήσετε το brew για να εγκαταστήσετε αυτό και άλλες εξαρτήσεις.

    brew install libomp

  2. Στη συνέχεια, εγκαταστήστε μια εφαρμογή BLAS; OpenBLAS είναι η προεπιλεγμένη επιλογή.

    brew install openblas

    Στη συνέχεια, εγκαταστήστε τα Xcode Command Line Tools.

    xcode-select --install
Windows

Στα Windows θα πρέπει να χρησιμοποιήσετε Anaconda3 ή Miniconda3 για να εγκαταστήσετε όλες τις εξαρτήσεις.

  1. Εγκατάσταση απαιτήσεων μεταγλωττιστή.

    conda install --update-deps vs2017_win-64 vs2017_win-32 msvc_runtime

Qiskit Aer είναι ένα πλαίσιο εξομοιωτή υψηλής απόδοσης για κβαντικά κυκλώματα. Παρέχει αρκετά backends για την επίτευξη διαφορετικών στόχων προσομοίωσης.

conda install --update-deps -c conda-forge -y openblas cmake

  1. Κατασκευάστε και εγκαταστήστε απευθείας το qiskit-aer

    Εάν έχετε εγκαταστήσει το pip <19.0.0 και το περιβάλλον σας δεν απαιτεί προσαρμοσμένη κατασκευή, εκτελέστε:

    cd qiskit-aer
pip install .

    Αυτό θα χτίσει την εφαρμογή και θα εγκαταστήσει το Aer.

    Εναλλακτικά, αν έχετε ένα νεότερο pip εγκατεστημένο ή αν έχετε κάποια προσαρμοσμένη απαίτηση, μπορείτε να φτιάξετε έναν Python wheel χειροκίνητα.

    cd qiskit-aer
python ./setup.py bdist_wheel

    Αν χρειαστεί να ορίσετε μια προσαρμοσμένη επιλογή κατά τη διάρκεια της κατασκευής του wheel, ανατρέξτε στο Προσαρμοσμένες επιλογές κατά την κατασκευή του wheel.

    Μετά την κατασκευή του Python wheel, θα αποθηκευτεί στον φάκελο dist/ στο αποθετήριο Aer. Η ακριβής έκδοση θα εξαρτηθεί

    cd dist
pip install qiskit_aer-*.whl

    Το ακριβές όνομα του αρχείου του wheel εξόδου εξαρτάται από την τρέχουσα έκδοση του Aer υπό ανάπτυξη.

Custom options

Το σύστημα κατασκευής Aer χρησιμοποιεί scikit-build για να εκτελέσει τη μεταγλώττιση κατά τη δημιουργία του με τη διεπαφή Python. Δρα ως διεπαφή για τα setuptools για να καλέσετε CMake και να μεταγλωττίσετε τις εφαρμογές για το τοπικό σας σύστημα.

Λόγω της πολυπλοκότητας της κατάρτισης των εφαρμογών, μπορεί να χρειαστεί να περάσετε τις επιλογές σε ένα ορισμένο μέρος της διαδικασίας κατασκευής. Ο τρόπος για να περάσετε μεταβλητές είναι:

python setup.py bdist_wheel [skbuild_opts] [-- [cmake_opts] [-- build_tool_opts]]

όπου τα στοιχεία μέσα στις αγκύλες [] είναι προαιρετικά και skbuild_opts, cmake_opts`, build_tool_opts πρέπει να αντικατασταθούν από σημαίες της επιλογής σας. Μια λίστα επιλογών CMake είναι διαθέσιμη εδώ: https://cmake.org/cmake/help/v3.6/manual/cmake.1.html#options. Για παράδειγμα, θα μπορούσατε να εκτελέσετε κάτι όπως:

python setup.py bdist_wheel -- -- -j8

Αυτό περνάει τη σημαία -j8 στο υποκείμενο σύστημα κατασκευής (το οποίο σε αυτή την περίπτωση είναι Automake), λέγοντάς του ότι θέλετε να οικοδομήσουμε παράλληλα χρησιμοποιώντας 8 διαδιακασίες.

Για παράδειγμα, μια κοινή περίπτωση χρήσης για αυτές τις σημαίες στο linux είναι να καθορίσετε μια συγκεκριμένη έκδοση του μεταγλωττιστή C++ που θα χρησιμοποιηθεί (κανονικά αν η προεπιλογή είναι πολύ παλιά):

python setup.py bdist_wheel -- -DCMAKE_CXX_COMPILER=g++-7

που θα πει στο CMake να χρησιμοποιεί την εντολή g++-7 αντί της προκαθορισμένης g++ κατά την μεταγλώττιση του Aer.

Μια άλλη συνηθισμένη περίπτωση χρήσης για αυτό, ανάλογα με το περιβάλλον σας, είναι ότι μπορεί να χρειαστεί να καθορίσετε το όνομα της πλατφόρμας σας και να απενεργοποιήσετε τη στατική σύνδεση.

python setup.py bdist_wheel --plat-name macosx-10.9-x86_64 \
-- -DSTATIC_LINKING=False -- -j8

Εδώ το --plat-name είναι μια σημαία στο setuptools, για να καθορίσετε το όνομα της πλατφόρμας που θα χρησιμοποιηθεί στα μεταδεδομένα του πακέτου, -DSTATIC_LINKING είναι μια σημαία για τη χρήση CMake για την απενεργοποίηση στατικής σύνδεσης, και -j8 είναι μια σημαία για τη χρήση του Automake για τη χρήση 8 διαδικασιών για μεταγλώττιση.

Installing Ignis from Source

  1. Κλωνοποιήστε το αποθετήριο του Ignis.

    git clone https://github.com/Qiskit/qiskit-ignis.git

  2. Η κλωνοποίηση του αποθετήριου δημιουργεί ένα τοπικό φάκελο με το όνομα qiskit-ignis.

    cd qiskit-ignis

  3. Αν θέλετε να εκτελέσετε δοκιμές ή linting ελέγχους, εγκαταστήστε τις απαιτήσεις για προγραμματιστές. Αυτό δεν απαιτείται για την εγκατάσταση ή τη χρήση του πακέτου qiskit-ignis για εγκατάσταση από την πηγή.

    pip install -r requirements-dev.txt

  4. Εγκαταστήστε το Ignis.

    pip install .

Αν θέλετε να το εγκαταστήσετε σε τροποποιήσιμη μορφή, η οποία σημαίνει ότι οι αλλαγές του κώδικα στο έργο δεν απαιτούν να εφαρμοστεί κάποια επανεγκατάσταση:

pip install -e .

    git clone https://github.com/Qiskit/qiskit-aqua.git

  2. Η κλωνοποίηση του αποθετήριου δημιουργεί ένα τοπικό φάκελο με το όνομα qiskit-aqua.

  3. Αν θέλετε να εκτελέσετε δοκιμές ή ελέγχους σύνδεσης, εγκαταστήστε τις απαιτήσεις για προγραμματιστές. Αυτό δεν απαιτείται για την εγκατάσταση ή χρήση του πακέτου qiskit-aqua κατά την εγκατάσταση από την πηγή.

    pip install -r requirements-dev.txt

  4. Εγκαταστήστε το Aqua.

    pip install .

Αν θέλετε να το εγκαταστήσετε σε τροποποιήσιμη μορφή, η οποία σημαίνει ότι οι αλλαγές του κώδικα στο έργο δεν απαιτούν να εφαρμοστεί κάποια επανεγκατάσταση:

pip install -e .

Installing IBM Quantum Provider from Source

  1. Κλωνοποιήστε το αποθετήριο του qiskit-ibmq-provider.

    git clone https://github.com/Qiskit/qiskit-ibmq-provider.git

  2. Η κλωνοποίηση του αποθετήριου δημιουργεί ένα τοπικό φάκελο με το όνομα `` qiskit-ibmq-provider ``.

    cd qiskit-ibmq-provider

  3. Αν θέλετε να εκτελέσετε δοκιμές ή linting ελέγχους, εγκαταστήστε τις απαιτήσεις για προγραμματιστές. Αυτό δεν απαιτείται για την εγκατάσταση ή χρήση του πακέτου qiskit-ibmq-provider κατά την εγκατάσταση από την πηγή.

    pip install -r requirements-dev.txt

  4. Εγκαταστήστε το qiskit-ibmq-provider.

    pip install .

Αν θέλετε να το εγκαταστήσετε σε τροποποιήσιμη μορφή, η οποία σημαίνει ότι οι αλλαγές του κώδικα στο έργο δεν απαιτούν να εφαρμοστεί κάποια επανεγκατάσταση:

pip install -e .

Instalacja Qiskit¶

Wymagania¶

Qiskit obsługuje Python 3.5 lub nowszy. Zarówno Python, jak i Qiskit są ewoluującymi ekosystemami i czasami, gdy pojawiają się w nich nowe wydania, mogą występować problemy z kompatybilnością.

Jeśli masz problemy z instalacją lub używaniem Qiskit po aktualizacji Pythona, sprawdź metadane pakietów Qiskit dla języka programowania, aby zobaczyć, które konkretne wersje Pythona są obsługiwane.

Zaleca się instalację ` Anaconda <https://www.anaconda.com/download/>` __, wieloplatformową dystrybucję Pythona do obliczeń naukowych. Jupyter, zawarty w pakiecie Anaconda, polecany jest do interakcji z Qiskit.

Qiskit jest testowany i obsługiwany na następujących 64-bitowych systemach:

  • Ubuntu 12.04 i nowsze

  • macOS 10.12.6 lub nowszy

  • Windows 7 lub nowszy

Instalacja¶

Zalecamy używanie wirtualnych środowisk Pythona do czystego oddzielenia Qiskit od innych aplikacji.

Najprostszym sposobem na korzystanie ze środowisk jest użycie komendy conda dołączonej do Anaconda. Środowisko Conda pozwala określić konkretną wersję Pythona i zestawu bibliotek. Otwórz okno terminala w katalogu, w którym chcesz pracować.

Preferowane jest użycie linii komend Anaconda zainstalowanego z Anaconda. Wszystko, co musisz zrobić, to stworzyć wirtualne środowisko wewnątrz Anaconda i aktywować środowisko. Te polecenia mogą być uruchamiane w wierszu poleceń Anaconda niezależnie od systemu Windows lub Linux.

Utwórz minimalne środowisko tylko z zainstalowanym w nim Pythonem.

conda create -n name_of_my_env python=3

Aktywuj nowe środowisko.

conda activate name_of_my_env

Informacja

W przypadku korzystania z wersji conda wcześniejszej niż 4.6, należy użyć: Dla systemu Linux lub macOS: ` ` source activate name_of_my_env ` ` Dla Windows: ` ` activate name_of_my_env ` `

Następnie zainstaluj pakiet Qiskit, w skład którego wchodzą Terra, Aer, Ignis i Aqua.

pip install qiskit

Informacja

Począwszy od Qiskit 0.13.0 pip 19 lub nowszy jest potrzebny, aby zainstalować qiskit-aer z prekompilowanego pliku binarnego w systemie Linux. Jeśli nie masz zainstalowanego pip 19, możesz uruchomić ` ` pip install -U pip ` `, aby go zaktualizować. Bez pip 19 lub nowszego ta komenda będzie próbował zainstalować qiskit-aer z sdist (dystrybucja źródła), które będą próbowały skompilować aer lokalnie.

Jeśli pakiety zostały zainstalowane poprawnie, możesz uruchomić conda list aby zobaczyć listę aktywnych pakietów w Twoim wirtualnym środowisku.

Jeśli zamierzasz korzystać z funkcji wizualizacji lub notebooków Jupyter, zaleca się zainstalować qiskit z dodatkami dla wizualizacji

pip install qiskit[visualization]

Warto zwrócić uwagę na to, że jeśli jesteś użytkownikiem zsh (który jest domyślną powłoką w nowszych wersjach macOS), musisz umieścić ` ` qiskit[visualization]` ` w cudzysłowach:

pip install 'qiskit[visualization]'

Informacja

Po zainstalowaniu i zweryfikowaniu pakietów Qiskit, które mają być używane, aby rozpocząć pracę, zaimportuj je do środowiska z językiem Python.

import qiskit

Informacja

Jeśli chcesz przyczynić się do rozwoju społeczności Qiskit, rozwijając i przyczyniając się do tworzenia najnowszych aktualizacji kodu Qiskit, zapoznaj się z Zbuduj pakiety Qiskit ze źródła.

Dostęp do systemu IBM Quantum¶

IBM Quantum offers several real quantum computers and high-performance classical computing simulators through its IBM Quantum Experience with Qiskit. Follow these steps to set up your Qiskit environment to send jobs to IBM Quantum systems.

Informacja

With the release of Qiskit 0.11, if you had previously saved your IBM Quantum Experience credentials locally, you might need to update them so that you can use the new IBM Quantum Experience V2. See Updating your IBM Quantum Experience Credentials.

Aby skonfigurować konto, należy utworzyć lokalny plik konfiguracyjny, który zawiera klucz API

1. Utwórz darmowe konto IBM Quantum Experience.

2. Przejdź do Moje konto, aby zobaczyć ustawienia konta.

3. Kliknij Skopiuj token aby skopiować token do schowka. Tymczasowo wklej ten token API do swojego ulubionego edytora tekstu, dzięki czemu możesz go później użyć, aby utworzyć plik konfiguracyjny konta.

4. Uruchom następujące polecenia, aby przechowywać swój token API lokalnie do późniejszego użycia w pliku konfiguracyjnym o nazwie qiskitrc. Zastąp MY_API_TOKEN wartością tokenu API, którą zapisałeś w edytorze tekstowym.

from qiskit import IBMQ
IBMQ.save_account('MY_API_TOKEN')

For more details, such as how to manage multiple IBM Quantum account credentials, refer to this tutorial titled The IBM Quantum Account.

Sprawdzenie zainstalowanej wersji¶

Ponieważ pakiet Qiskit składa się z wielu elementów, proste wyświetlenie wersji używając qiskit.__version__ może wprowadzać w błąd, ponieważ instrukcja zwraca tylko wersję pakietu qiskit-terra. Dzieje się tak, ponieważ przestrzeń nazw qiskit w Pythonie nie pochodzi z pakietu Qiskit, a zamiast tego jest częścią pakietu qiskit-terra.

import qiskit
qiskit.__version__

'0.15.1'

Aby zobaczyć wersje wszystkich elementów Qiskit w danym środowisku, można użyć atrybutu ` __qiskit_version__ `. Przykładowo uruchomienie następującej komendy spowoduje wyświetlenie słownika, który zawiera wersje dla każdego z zainstalowanych pakietów Qiskit.

qiskit.__qiskit_version__

{'qiskit-terra': '0.15.1',
 'qiskit-aer': '0.6.1',
 'qiskit-ignis': '0.4.0',
 'qiskit-ibmq-provider': '0.8.0',
 'qiskit-aqua': '0.7.5',
 'qiskit': '0.20.0'}

Wskazówka

Jeśli zgłaszasz problem lub musisz udostępnić informacje o zainstalowanej wersji Qiskit, użyj atrybutu __qiskit_version__`.

Pierwsze kroki z Qiskit¶

Na działanie Qiskit składają się trzy główne etapy:

  • Budowanie: projekt obwodu kwantowego, który reprezentuje rozważany problem.

  • Wykonanie: uruchomienie eksperymentów na różnych platformach (ang. backends) (które zawierają zarówno systemy jak i symulatory).

  • Analiza: obliczanie statystyk podsumowujących oraz wizualizacja rezultatów eksperymentów.

Poniżej przedstawiono przykład całościowego działania, z każdym krokiem objaśnionym szczegółowo w kolejnych sekcjach:

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  Aer)
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = Aer.get_backend('qasm_simulator')

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# Execute the circuit on the qasm simulator
job = execute(circuit, simulator, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()


Total count for 00 and 11 are: {'00': 491, '11': 509}

     ┌───┐     ┌─┐   
q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1 
# Plot a histogram
plot_histogram(counts)

Działanie krok po kroku¶

Powyższy program może być podzielony na sześć etapów:

  1. Zaimportowanie pakietów

  2. Inicjalizacja zmiennych

  3. Dodawanie bramek

  4. Wizualizacja obwodu

  5. Symulacja eksperymentu

  6. Wizualizacja wyników

Krok 1: Importowanie pakietów¶

Podstawowe elementy wymagane przez Twój program są importowane następująco:

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  Aer)
from qiskit.visualization import plot_histogram

Bardziej szczegółowo importowanie to:

  • QuantumCircuit: może być traktowany jako instrukcje do systemu kwantowego. Zawiera wszystkie Twoje operacje kwantowe.

  • execute: uruchamia Twój obwód / eksperyment.

  • Aer: obsługuje platformy symulatora.

  • plot_histogram: tworzy histogramy.

Krok 2: Inicjowanie zmiennych¶

Spójrzmy na następującą linię kodu

circuit = QuantumCircuit(2, 2)

Inicjujemy dwa kubity w stanie zero; z dwoma klasycznymi bitami ustawionymi na zero; circuit jest obwodem kwantowym.

Składnia:

  • QuantumCircuit(int, int)

Krok 3: Dodawanie bramek¶

Bramki (operacje) mogą być dodawane w celu zmiany rejestrów w Twoim obwodzie.

Spójrzmy na następujące trzy linie kodu:

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

Bramki są dodawane do obwodu jedna po drugiej, żeby utworzyć stan Bella

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

Powyższy kod dotyczy następujących bramek:

  • QuantumCircuit.h(0): Bramka Hadamarda \(H\) dla kubitu 0, która ustawia go w stan superpozycji.

  • QuantumCircuit.cx(0, 1): Operacja kontrolowanej negacji (\(C_{X}\)) dla kubitu kontrolnego 0 oraz docelowego kubitu 1, ustawiająca kubity w stan splątany.

  • QuantumCircuit.measure([0,1], [0,1]): jeśli przekażesz wszystkie kwantowe i klasyczne rejestry do measure, wtedy wynik pomariu i-tego kubitu zostanie zapisany w i-tym klasycznym bicie.

Krok 4: Wizualizacja obwodu¶

Możesz użyć qiskit.circuit.QuantumCircuit.draw() do różnorakiej formy wizualizacji obwodu, i wykorzystać w zależności od formy używanej w podręcznikach i artykułach badawczych.

circuit.draw()

     ┌───┐     ┌─┐   
q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

W tym obwodzie, qubity są pokazywane z qubit zero u góry i qubit jeden u dołu. Obwód jest czytany od lewej do prawej, co oznacza, że bramki, które są stosowane wcześniej w obwodzie, umieszczane są od lewej do prawej.

Domyślnym wyjściem dla ` `QuantumCircuit.draw () ` ` lub ` `qiskit.visualization.circuit_drawer () ` ` jest wyjście tekstowe. Jednak w zależności od środowiska lokalnego, można zmienić te wartości domyślne na bardziej odpowiednie dla danego przypadku użycia. Jest to wykonywane przy użyciu pliku konfiguracyjnego użytkownika. Domyślnie plik konfiguracyjny użytkownika powinien znajdować się w pliku ` `~/.qiskit/settings.conf ` ` ` i jest plikiem ` `.ini ` `.

Na przykład, plik settings.conf do ustawień wyjścia Matplotlib to:

[default]
circuit_drawer = mpl

Możesz użyć dowolnego z dostępnych formatów wyjściowych jako wartości dla tej konfiguracji, w tym text, mpl, latex, i latex_source.

Krok 5: Symulacja eksperymentu¶

Qiskit Aer jest wysokowydajnym środowiskiem symulatorów dla obwodów kwantowych. Zapewnia on ` kilka platform <apidoc/providers/aer/aer_backends.html>` __, dzięki którym można osiągnąć różne cele symulacji.

Jeśli masz problemy z instalacją Aer, możesz alternatywnie użyć symulatora Basic Aer, zastępując ` Aer ` ` z ` BasicAer `. Basic Aer jest dołączany do Qiskit Terra.

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  BasicAer)
...

Aby zasymulować ten obwód, należy użyć ` ` qasm_simulator ` `. Każde uruchomienie tego obwodu będzie skutkować uzyskaniem wyniku w formie ciągu bitów o numerze 00, albo 11.

simulator = Aer.get_backend('qasm_simulator')
job = execute(circuit, simulator, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'00': 463, '11': 537}

Zgodnie z oczekiwaniami, wartość dla ciąg bitów wyjściowych 00 wynosi około 50 %. Liczba uruchomień obwodu może być określona za pomocą argumentu shots metody execute. Liczba powtórzeń symulacji została ustawiona na 1000 (domyślnie 1024).

Gdy już masz obiekt result, to możesz teraz uzyskać dostęp do liczników za pomocą metody get_counts (circuit). To daje dostęp zagregowanego wyniku eksperymentu, który został przeprowadzony.

Krok 4: Wizualizacja wyników¶

Qiskit zapewnia wiele form wizualizacji,

wliczając funkcję plot_histogram, aby wyświetlić wyniki.

plot_histogram(counts)

Obserwowane prawdopodobieństwo \(Pr(00)\) i \(Pr(11)\) oblicza się, przyjmując odpowiednią liczbę symulacji i dzieląc się przez całkowitą liczbę symulacji.

Informacja

Spróbuj zmienić słowo kluczowe shots w metodzie execute aby zobaczyć, jak zmieniają się szacowane wartości prawdopodobieństwa.

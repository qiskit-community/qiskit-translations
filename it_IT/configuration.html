Configurazione Locale¶

Una volta che Qiskit è stato installato ed è funzionante, è possibile seguire alcuni passaggi di configurazione opzionali per modificare alcune impostazioni di default di Qiskit, in modo da adattarlo a casi d’uso specifici.

File User Config¶

La maggior parte delle configurazioni locali di Qiskit si trovano nel file user config. Il file, in formato ini, può essere utilizzato per modificare le impostazioni di defalt in Qiskit.

Ad esempio:

[default]
circuit_drawer = mpl
circuit_mpl_style = default
package_warnings = False
transpile_optimization_level=3

La posizione predefinita di questo file è ~/.qiskit/settings.conf, ma il path può essere modificato utilizzando la variabile di ambiente QISKIT_SETTINGS. Se a QISKIT_SETTINGS viene dato un valore, questo verrà utilizzato come path del file user config.

Opzioni disponibili:

  • circuit_drawer: Viene utilizzato per modificare il backend di default del circuit drawer qiskit.circuit.QuantumCircuit.draw() e qiskit.visualization.circuit_drawer(). Può essere impostato su latex, mpl, text o latex_source, e quando il kwarg output non è esplicitamente indicato verrà utilizzato il backend qui impostato.

  • circuit_mpl_style: Indica lo stile di default utilizzato per il backend di output mpl del circuit drawer qiskit.circuit.QuantumCircuit.draw() e qiskit.visualization.circuit_drawer(). Può essere impostato a default o bw.

  • package_warnings: È un flag booleano che se impostato a true nasconderà i RuntimeWarnings che vengono normalmente generati quando o qiskit-aer o qiskit-ibmq-provider non vengono trovati.

  • transpile_optimization_level: Il parametro accetta integer tra 0-3 ed è utilizzato per modificare l’optimization level di default per transpile() e execute().

Variabili di Ambiente¶

Ci sono anche alcune variabili di ambiente che possono essere impostate per modificare il comportamento predefinito di Qiskit.

  • QISKIT_IN_PARALLEL: se questa variabile viene impostata a qualsiasi valore diverso da FALSE verrà disabilitato l’utilizzo del multiprocessing utilizzato per parallelizzare alcune operazioni (come la transpilation su più circuiti) in Qiskit Terra.

  • QISKIT_SUPPRESS_PACKAGING_WARNINGS: Quando impostato a Y, verrà soppresso il RuntimeWarnings che viene generato quando qiskit-aer o qiskit-ibmq-provider non sono trovati.

  • QISKIT_IBMQ_PROVIDER_LOG_LEVEL: Indica il livello di log da utilizzare per il modulo qiskit-ibmq-provider. Se viene impostato un livello non valido, viene utilizzato quello di default che è WARNING. I possibili valori per i livello di log sono DEBUG, INFO, WARNING, ERROR, e CRITICAL (case-insensitive). Se la variabile di ambiente non viene valorizzata, allora viene utilizzato il livello di log del parent logger, il cui default è sempre WARNING.

  • QISKIT_IBMQ_PROVIDER_LOG_FILE: Indica il nome del file di log da utilizzare per i messaggi di log originati da qiskit-ibmq-provider. Se specificato, i messaggi verranno indirizzati solo nel file. Altrimenti i messaggi di log saranno indirizzati nello standard error (in genere a schermo).

  • QISKIT_AQUA_MAX_GATES_PER_JOB: Un parametro opzionale che serve ad impostare una soglia utilizzata per suddividere i diversi circuiti generati da Aqua in più job sottomessi ad un backend, in base al numero di gate utilizzati.

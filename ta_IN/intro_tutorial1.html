Qiskit அறிமுகம்¶

Qiskit பயன்படுத்தும் போது ஒரு பயனர் பணிப்பாய்வு பெயரளவில் நான்கு உயர் மட்ட படிகளைப் பின்பற்றுகிறது:

  • Build: நீங்கள் கருத்தில் கொள்ளும் சிக்கலைக் குறிக்கும் குவாண்டம் சர்க்யூட்(கள்) வடிவமைக்கவும்.

  • Compile: ஒரு குறிப்பிட்ட குவாண்டம் சேவைக்கான சுற்றுகளைத் தொகுக்கவும், எ.கா. குவாண்டம் அமைப்பு அல்லது கிளாசிக்கல் சிமுலேட்டர்.

  • ** Run **: குறிப்பிட்ட குவாண்டம் சேவையில் (களில்) தொகுக்கப்பட்ட சுற்றுகளை இயக்கவும். இந்த சேவைகள் மேகக்கணி சார்ந்தவை அல்லது உள்ளூர்.

  • Analyze: சுருக்கபுள்ளிவிவரங்களைக் கணக்கிடு மற்றும் சோதனைகளின் முடிவுகளை கற்பனை செய்யுங்கள்.

ஒவ்வொரு அடியும் அடுத்தடுத்த பிரிவுகளில் விரிவாக விளக்கப்பட்டுள்ள நிலையில், முழு பணிப்பாய்வின் ஒரு உதாரணம் இங்கே:

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = QasmSimulator()

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# compile the circuit down to low-level QASM instructions
# supported by the backend (not needed for simple circuits)
compiled_circuit = transpile(circuit, simulator)

# Execute the circuit on the qasm simulator
job = simulator.run(compiled_circuit, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()

பணிப்பாய்வு படி படியாக¶

மேலே உள்ள நிரலை ஆறு படிகளாக உடைக்கலாம்:

  1. பொதிகளை இறக்குமதி செய்

  2. மாறிகளை துவக்கு

  3. கேட்களைச் சேர்க்கவும்

  4. circuitகளை காட்சிப்படுத்தவும்

  5. பரிசோதனையை உருவகப்படுத்தவும்

  6. முடிவுகளை காட்சிப்படுத்து

Step 1: தொகுப்புகளை இறக்குமதி செய்க¶

உங்கள் நிரலுக்கு தேவையான அடிப்படை கூறுகள் பின்வருமாறு இறக்குமதி செய்யப்படுகின்றன:

import numpy as np
from qiskit import QuantumCircuit
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

இன்னும் விரிவாக, இறக்குமதிகள்

  • QuantumCircuit: குவாண்டம் அமைப்பின் அறிவுறுத்தல்களாக கருதப்படலாம். இது உங்கள் குவாண்டம் செயல்பாடுகளை வைத்திருக்கிறது.

  • `` QasmSimulator``: ஏர் உயர் செயல்திறன் சுற்று சிமுலேட்டர்.

  • 'plot_histogram': ஹிஸ்டோகிராம்களை உருவாக்குகிறது.

படி 2: மாறிகளை துவக்கு¶

குறியீட்டின் அடுத்த வரியைக் கவனியுங்கள்

circuit = QuantumCircuit(2, 2)

இங்கே, நீங்கள் பூஜ்ஜிய நிலையில் 2 குபிட்களுடன் துவக்குகிறீர்கள்; 2 கிளாசிக்கல் பிட்கள் பூஜ்ஜியமாக அமைக்கப்பட்டுள்ளது; மற்றும் 'சுற்று' என்பது குவாண்டம் சுற்று ஆகும்.

சொற்றொடரியல்:

  • QuantumCircuit(int, int)

படி 3 : கேட்களை சேர்¶

உங்கள் சுற்று பதிவேடுகளை கையாள வாயில்கள் (செயல்பாடுகள்) சேர்க்கலாம்.

பின்வரும் மூன்று குறியீடு வரிகளைக் கவனியுங்கள்:

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

கேட்கள் ஒன்று சேர்க்கப்பட்டு ஒன்று-ஒன்று மூலம் பெல் ஸ்டேடை வடிவமைக்கப் படுகிறது

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

மேலே குறியீடு பின்வரும் கேட்களை பயன்படுத்துகிறது:

  • QuantumCircuit.h(0): A Hadamard gate \(H\) இது குபிட் 0 ஐ superposition state வைக்கிறது.

  • "QuantumCircuit.cx(0, 1)": கட்டுப்பாட்டு க்யூபிட் 0 மற்றும் இலக்கு குபிட் 1 இல் ஒரு கட்டுப்படுத்தப்பட்ட-இல்லை செயல்பாடு (\(CNOT\)) ஒரு **entangled state**இல் குபிட்களை வைக்கிறது.

  • "QuantumCircuit.measure([0,1], [0,1])": நீங்கள் முழு குவாண்டம் மற்றும் கிளாசிக்கல் பதிவேடுகளை "measure" கடந்து சென்றால், ஐத் க்யூபிட்டின் அளவீட்டு முடிவு ஐத் கிளாசிக்கல் பிட்டில் சேமிக்கப்படும்.

Step 4: மின்சுற்று காண்க¶

நீங்கள் பயன்படுத்த முடியும் :meth:'qiskit.circuit.QuantumCircuit.draw' நீங்கள் பல பாடப்புத்தகங்கள் மற்றும் ஆராய்ச்சி கட்டுரைகள் பயன்படுத்தப்படும் பல்வேறு வடிவங்களில் வடிவமைக்கப்பட்டுள்ளது என்று சுற்று பார்க்க.

circuit.draw()

     ┌───┐     ┌─┐   
q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

இச்சுற்றில், க்யூபிட்கள் மேலே க்யூபிட் பூஜ்ஜியத்துடனும், கீழே உள்ள ஒன்றை க்யூபிட் செய்யவும் உத்தரவிடப்படுகின்றன. சுற்று இடது-க்கு-வலது படிக்கப்படுகிறது, அதாவது சுற்றில் முன்பு பயன்படுத்தப்படும் வாயில்கள் இடது புறம் இன்னும் அதிகமாக க் காட்டுகின்றன.

'QuantumCircuit.draw()' அல்லது 'qiskit.visualization.circuit_drawer()' க்கான இயல்புநிலை பின்முனை உரை பின்முனையாகும். இருப்பினும், உங்கள் உள்ளூர் சூழலைப் பொறுத்து, இந்த இயல்புநிலைகளை உங்கள் பயன்பாட்டு வழக்குக்கு மிகவும் பொருத்தமானதாக மாற்ற விரும்பலாம். இது பயனர் உள்வடிவமைப்பு கோப்புடன் செய்யப்படுகிறது. முன்னிருப்பாக பயனர் உள்வடிவமைப்பு கோப்பு '~/.qiskit/settings.conf' இல் இருக்க வேண்டும் மற்றும் '.ini' கோப்பாக இருக்க வேண்டும்.

உதாரணமாக, settings.conf மேட்ப்ளாட்லிப் வரைபடம் அமைப்பதற்கான கோப்பு:

[default]
circuit_drawer = mpl

இந்த உட்சேர்க்கைக்கான மதிப்பாக செல்லுபடியாகும் சர்க்யூட் டிராயரை நீங்கள் பயன்படுத்தலாம், இதில் உரை, எம். பி. எல், லேடெக்ஸ் மற்றும் latex_source அடங்கும்.

படி 5 : பரிசோதனையை உருவகப்படுத்தவும்¶

Qiskit Aer குவாண்டம் சுற்றுகளுக்கான உயர் செயல்திறன் போலி கட்டமைப்பாகும். இது வெவ்வேறு உருவகப்படுத்துதல் இலக்குகளை அடைய 'several backends <apidoc/providers/aer/aer_backends.html>'__ வழங்குகிறது.

Aer ஐ நிறுவுவதில் சிக்கல் இருந்தால், Aer ஐ BasicAer உடன் மாற்றுவதன் மூலம் நீங்கள் அடிப்படை Aer வழங்குநரைப் பயன்படுத்தலாம். Qiskit டெர்ராவில் அடிப்படை Aer சேர்க்கப்பட்டுள்ளது.

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.basicaer import QasmSimulatorPy
...

இந்த சுற்று உருவகப்படுத்த, நீங்கள்"'qasm_simulator" பயன்படுத்த வேண்டும். இந்த சுற்றின் ஒவ்வொரு ஓட்டமும் பிட் சரம் 00 அல்லது 11 ஐ கொடுக்கும்.

simulator = QasmSimulator()
compiled_circuit = transpile(circuit, simulator)
job = simulator.run(compiled_circuit, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'00': 503, '11': 497}

எதிர்பார்த்தபடி, வெளியீடு பிட் சரம் நேரம் சுமார் 50 சதவீதம் ஆகும். சுற்று எத்தனை முறை இயங்குகிறது என்பதை ''சுட்டிகள்' என்ற ``execute``முறையின் வாதம் மூலம் குறிப்பிடலாம். உருவகப்படுத்துதல் காட்சிகளின் எண்ணிக்கை 1000 ஆக அமைக்கப்பட்டது (இயல்புநிலை 1024).

நீங்கள் ஒரு 'result' பொருள் இருந்தால், நீங்கள் முறை மூலம் எண்ணிக்கைகளை அணுக முடியும்``get_counts(circuit)``. இது நீங்கள் நடத்திய சோதனையின் ஒட்டுமொத்த விளைவுகளை உங்களுக்கு வழங்குகிறது.

படி முறை 6 : முடிவுகளை காட்சிப்படுத்து¶

Qiskit 'பல காட்சிப்படுத்தல்களை வழங்குகிறது <apidoc/visualization.html>'__,

உங்கள் முடிவுகளைக் காண, "plot_histogram" என்ற செயல்பாடு உட்பட.

plot_histogram(counts)

கவனிக்கப்பட்ட நிகழ்தகவுகள் :math:'Pr(00)' மற்றும் :math:'Pr(11)' ஆகியவை அந்தந்த எண்ணிக்கைகளை எடுத்து மொத்த காட்சிகளின் எண்ணிக்கையால் வகுக்கப்படுகின்றன.

Note

மதிப்பிடப்பட்ட நிகழ்தகவுகள் எவ்வாறு மாறுகின்றன என்பதைப் பார்க்க execute` முறையில் ``shots முக்கிய வார்த்தையை மாற்ற முயற்சிக்கவும்.

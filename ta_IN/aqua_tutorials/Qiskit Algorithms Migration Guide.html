Qiskit வழிமுறைகள் இடம்பெயர்வு வழிகாட்டி¶

செயலிகளை மறுசீரமைத்தல்

Qiskit 0.25.0 வெளியீட்டில் பயன்பாடுகள் மற்றும் வழிமுறைகளின் மறுசீரமைப்பு அடங்கும். முன்பு Qiskit அக்வா என குறிப்பிடப்பட்டவை, Qiskit-டின் ஒற்றை பயன்பாடுகள் மற்றும் வழிமுறைகள் தொகுதி, இப்போது உகப்பாக்கம், நிதி, இயந்திர கற்றல் மற்றும் இயற்கை (இயற்பியல் மற்றும் வேதியியல் உட்பட) ஆகியவற்றிற்கான பிரத்யேக பயன்பாட்டு தொகுதிகளாக பிரிக்கப்பட்டுள்ளது. முக்கிய வழிமுறைகள் மற்றும் ஆப்ஃப்ளோ ஆபரேட்டர் செயல்பாடு Qiskit டெர்ராவுக்கு நகர்த்தப்படுகின்றன.

வழிமுறைகள் இடைமுகங்கள்

மறுசீரமைப்பிற்கு கூடுதலாக, அனைத்து வழிமுறைகளும் ஒரு புதிய ஒருங்கிணைந்த முன்னுதாரணத்தைப் பின்பற்றுகின்றன: வழிமுறைகள் அவை தீர்க்கும் சிக்கல்களுக்கு ஏற்ப வகைப்படுத்தப்படுகின்றன, மேலும் ஒரு பயன்பாட்டு வகுப்பு வழிமுறைகள் ஒரே சிக்கலைத் தீர்க்க மாறி மாறிப் பயன்படுத்தலாம். இதன் பொருள், முன்பு போலல்லாமல், வழிமுறை நிகழ்வுகள் அவை தீர்க்கும் சிக்கலில் இருந்து துண்டிக்கப்படுகின்றன. இதை ஒரு வரைபடத்தில் சுருக்கமாகக் கூறலாம்:

எடுத்துக்காட்டாக, மாறுபட்ட குவாண்டம் ஐகென்சால்வர், VQE என்பது ஒரு ஆபரேட்டரின் குறைந்தபட்ச ஐகென்வேல்யூவைக் கணக்கிடுகையில் MinimumEigensolver ஆகும். இங்கே சிக்கல் ஆபரேட்டருடன் குறிப்பிடப்பட்டுள்ளது, அதன் ஐகென்வேல்யூவை நாங்கள் தேடுகிறோம், அதே நேரத்தில் மாறுபட்ட அன்சாட்ஸ் சர்க்யூட் மற்றும் கிளாசிக்கல் ஆப்டிமைசர் போன்ற பண்புகள் வழிமுறையின் பண்புகள். அதாவது VQE பின்வரும் கட்டமைப்பைக் கொண்டுள்ளது

vqe = VQE(ansatz, optimizer)
result = vqe.compute_minimum_eigenvalue(operator)

உங்கள் ஆபரேட்டரின் சமமான மதிப்புகளைக் கணக்கிடுவதற்கு MinimumEigensolver இடைமுகத்தை செயல்படுத்தும் வேறு எந்த வழிமுறையுடனும் VQE ஐ பரிமாறிக்கொள்ளலாம், எ.கா.

numpy_based = NumPyMinimumEigensolver()
classical_reference = numpy_based.compute_minimum_eigenvalue(operator)

இது வெவ்வேறு வழிமுறைகளுக்கு இடையில் எளிதாக மாறவும், கிளாசிக்கல் குறிப்புகளுக்கு எதிராக சரிபார்க்கவும், உங்கள் சொந்த செயலாக்கத்தை வழங்கவும் உங்களை அனுமதிக்கிறது \(-\) நீங்கள் ஏற்கனவே இருக்கும் இடைமுகத்தை செயல்படுத்த வேண்டும்.

Qiskit அக்வாவைப் பயன்படுத்தி உங்கள் தற்போதைய குறியீட்டை புதிய கட்டமைப்பிற்கு மாற்றுவதற்கு இந்த நோட்புக் இடம்பெயர்வு வழிகாட்டியாக செயல்படுகிறது.

இந்த நோட்புக்கிற்கான நீக்குதல் எச்சரிக்கையை நாங்கள் முடக்குகிறோம், எனவே qiskit.aqua இலிருந்து ஒரு பொருளை உடனடியாக நிறுவும்போது நீங்கள் எதையும் பார்க்க மாட்டீர்கள். இருப்பினும், முழு தொகுப்பும் நீக்கப்பட்டது மற்றும் பின்வருவது போன்ற எச்சரிக்கையை வெளியிடும் என்பதை நினைவில் கொள்க:

from qiskit import Aer

backend = Aer.get_backend('statevector_simulator')
aqua_qinstance = AquaQuantumInstance(backend, seed_simulator=2, seed_transpiler=2)

புதியது:

from qiskit import Aer
from qiskit.utils import QuantumInstance

backend = Aer.get_backend('statevector_simulator')

ஆபரேட்டர்கள்¶

ஒப்ஃப்ளோ ஆபரேட்டர்கள் நகர்ந்தனர்

qiskit.aqua.operators

to

qiskit.opflow

முன்பு:

[5]:

from qiskit.aqua.operators import X, I, Y

op = (X ^ I) + (Y ^ 2)

புதியது:

[6]:

from qiskit.opflow import X, I, Y

op = (X ^ I) + (Y ^ 2)

கூடுதல் அம்சங்கள்:

Qiskit.opflow உடன், Pauli ஸ்ட்ரிங்களின் தொகைகளின் புதிய, திறமையான பிரதிநிதித்துவத்தை நாங்கள் அறிமுகப்படுத்துகிறோம், இது Paulis-ன் மிகப் பெரிய தொகைகளில் கணக்கீடுகளை கணிசமாக விரைவுபடுத்துகிறது. Pauli ஸ்ட்ரிங்களை சுருக்கமாகக் கூறினால் இந்த திறமையான பிரதிநிதித்துவம் தானாகவே பயன்படுத்தப்படுகிறது:

[7]:

op = (X ^ X ^ Y ^ Y) + (X ^ 4) + (Y ^ 4) + (I ^ X ^ I ^ I)
type(op)

சுருக்கம்¶

முந்தைய அமைப்பு

grover = Grover(oracle_settings, algorithm_settings)
result = grover.run()

பிரித்தல் சிக்கல் / ஆரக்கிள் அமைப்புகள் மற்றும் வழிமுறை அமைப்புகளுக்கு மாற்றப்பட்டுள்ளது

இடம்பெயர்வு வழிகாட்டி¶

சர்க்யூட்களாக வழங்கப்பட்ட ஆரக்கிள்களுக்கும், நல்ல நிலைகளைத் தீர்மானிக்க is_good_state செயல்பாட்டிற்கும்

[10]:

from qiskit.circuit import QuantumCircuit

oracle = QuantumCircuit(2)
oracle.cz(0, 1)

def is_good_state(bitstr):
    return sum(map(int, bitstr)) == 2

முன்பு:

[11]:

from qiskit.aqua.algorithms import Grover

grover = Grover(oracle, is_good_state, quantum_instance=aqua_qinstance)
result = grover.run()
print('Top measurement:', result.top_measurement)

Top measurement: 11

புதியது:

[12]:

from qiskit.algorithms import Grover, AmplificationProblem

problem = AmplificationProblem(oracle=oracle, is_good_state=is_good_state)
grover = Grover(quantum_instance=qinstance)
result = grover.amplify(problem)
print('Top measurement:', result.top_measurement)

Top measurement: 11

QuantumCircuit வகுப்பை அடிப்படை பழமையானதாகப் பயன்படுத்துவதற்கான அனைத்து வழிமுறைகளையும் நாங்கள் நெறிப்படுத்துகிறோம் என்பதால், qiskit.aqua.compontents.Oracle வகுப்பைப் பயன்படுத்தி ஆரக்கிள்களை வரையறுக்கிறோம், எ.கா. LogicalExpressionOracle பயன்படுத்துவதற்கு பதிலாக இப்போது நீங்கள் சர்க்யூட் நூலகத்திலிருந்து PhaseOracle சர்க்யூட் பயன்படுத்தலாம்.

முன்பு:

[13]:

from qiskit.aqua.components.oracles import LogicalExpressionOracle
from qiskit.aqua.algorithms import Grover

oracle = LogicalExpressionOracle('x & ~y')
grover = Grover(oracle, quantum_instance=aqua_qinstance)
result = grover.run()
print('Top measurement:', result.top_measurement)

Top measurement: 01

புதியது:

[14]:

from qiskit.circuit.library import PhaseOracle
from qiskit.algorithms import Grover, AmplificationProblem

oracle = PhaseOracle('x & ~y')
problem = AmplificationProblem(oracle=oracle, is_good_state=oracle.evaluate_bitstring)
grover = Grover(quantum_instance=qinstance)
result = grover.amplify(problem)
print('Top measurement:', result.top_measurement)

Top measurement: 11

qiskit.aqua.components.oracles.TruthTableOracle இன்னும் பொருத்தப்ப்படுத்தப்படவில்லை, ஆனால் qiskit.circuit.classicalfunction தொகுதி மூலம் நடத்தை எளிதில் அடைய முடியும், க்ரோவரின் வழிமுறையின் பயிற்சிகளைப் பார்க்கவும்.

மேலும் எடுத்துக்காட்டுகள்¶

சர்க்யூட்டை உருவாக்க நாம் construct_circuit என்று அழைக்கலாம் மற்றும் நாங்கள் ஆர்வமாக உள்ள சிக்கல் நிகழ்வை அனுப்பலாம்:

[15]:

power = 2
grover.construct_circuit(problem, power).draw('mpl', style='iqx')

[15]:

அலைவீச்சு மதிப்பீடு¶

சுருக்கம்¶

அனைத்து அலைவீச்சு மதிப்பீட்டு வழிமுறைகளுக்கும் * AmplitudeEstimation * IterativeAmplitudeEstimation * MaximumLikelihoodAmplitudeEstimation, மற்றும் * FasterAmplitudeEstimation

இடைமுகம் மாற்றப்பட்டது

qae = AmplitudeEstimation(algorithm_settings, estimation_settings)
result = qae.run()

சிக்கல் / ஆரக்கிள் அமைப்புகள் மற்றும் வழிமுறை அமைப்புகளைப் பிரிக்க

qae = AmplitudeEstimation(algorithm_settings)
problem = EstimationProblem(oracle_settings)
result = qae.amplify(problem)

இடம்பெயர்வு வழிகாட்டி¶

இங்கே, எங்கள் ஒற்றை க்யூபிட்-ல் \(|1\rangle\) ஐ அளவிடுவதற்கான நிகழ்தகவை மதிப்பிட விரும்புகிறோம். நிலை தயாரிப்பு சர்க்யூட் என வழங்கப்பட்டால்

[16]:

import numpy as np

probability = 0.25
rotation_angle = 2 * np.arcsin(np.sqrt(probability))

state_preparation = QuantumCircuit(1)
state_preparation.ry(rotation_angle, 0)

objective_qubits = [0]  # the good states are identified by qubit 0 being in state |1>

print('Target probability:', probability)

state_preparation.draw(output='mpl', style='iqx')

Target probability: 0.25

[16]:

முன்பு:

[17]:

from qiskit.aqua.algorithms import AmplitudeEstimation

# instantiate the algorithm and passing the problem instance
ae = AmplitudeEstimation(3, state_preparation, quantum_instance=aqua_qinstance)

# run the algorithm
result = ae.run()

# print the results
print('Grid-based estimate:', result.estimation)
print('Improved continuous estimate:', result.mle)

Grid-based estimate: 0.1464466
Improved continuous estimate: 0.2499999956539467

இப்போது:

[18]:

from qiskit.algorithms import AmplitudeEstimation, EstimationProblem

problem = EstimationProblem(state_preparation=state_preparation, objective_qubits=objective_qubits)
ae = AmplitudeEstimation(num_eval_qubits=3, quantum_instance=qinstance)

result = ae.estimate(problem)
print('Grid-based estimate:', result.estimation)
print('Improved continuous estimate:', result.mle)

Grid-based estimate: 0.1464466
Improved continuous estimate: 0.2499999956513743

குறைந்தபட்ச ஐகென்வேல்யூஸ்¶

சுருக்கம்¶

  • இடைமுகம் பெரும்பாலும் ஒரே மாதிரியாகவே இருந்தது, ஆனால் முன்பு துவக்கத்தில் ஆபரேட்டரை அனுப்ப முடிந்தது.

  • ஆபரேட்டர்கள் இப்போது qiskit.aqua.operators க்கு பதிலாக qiskit.opflow இலிருந்து ஆபரேட்டர்களுடன் கட்டமைக்கப்பட வேண்டும்.

  • VQE ஆர்கியுமென்ட்டான var_form, ansatz ஆக மறுபெயரிடப்பட்டது.

HHL¶

சுருக்கம்¶

ஒரு உள்ளுணர்வு இடைமுகத்தை அனுமதிக்க மற்றும் முடிவின் திறமையான, சர்க்யூட் அடிப்படையிலான பிரதிநிதித்துவத்தை வழங்க HHL முற்றிலும் மறுசீரமைக்கப்பட்டுள்ளது.

இடம்பெயர்வு வழிகாட்டி¶

பின்வரும் நேரியல் முறையை தீர்க்க விரும்புகிறோம் என்று வைத்துக் கொள்ளுங்கள்

\[\begin{split}\begin{pmatrix} 1 & -1/3 \\ -1/3 & 1 \\ \end{pmatrix} \vec x = \begin{pmatrix} 1 \\ 0 \end{pmatrix}\end{split}\]
[42]:

import numpy as np

matrix = np.array([[1, -1/3], [-1/3, 1]])
vector = np.array([1, 0])

முன்பு:

[43]:

from qiskit.circuit.library import QFT
from qiskit.aqua.algorithms import HHL
from qiskit.aqua.components.eigs import EigsQPE
from qiskit.aqua.components.reciprocals import LookupRotation
from qiskit.aqua.components.initial_states import Custom
from qiskit.aqua.operators import MatrixOperator

def create_eigs(matrix, num_auxiliary, num_time_slices, negative_evals):
    ne_qfts = [None, None]
    if negative_evals:
        num_auxiliary += 1
        ne_qfts = [QFT(num_auxiliary - 1), QFT(num_auxiliary - 1).inverse()]
    return EigsQPE(MatrixOperator(matrix=matrix),
                   QFT(num_auxiliary).inverse(),
                   num_time_slices=num_time_slices,
                   num_ancillae=num_auxiliary,
                   expansion_mode='suzuki',
                   expansion_order=2,
                   evo_time=None,
                   negative_evals=negative_evals,
                   ne_qfts=ne_qfts)

orig_size = len(vector)

matrix, vector, truncate_powerdim, truncate_hermitian = HHL.matrix_resize(matrix, vector)

# Initialize eigenvalue finding module
eigs = create_eigs(matrix, 3, 50, False)
num_q, num_a = eigs.get_register_sizes()

# Initialize initial state module
init_state = Custom(num_q, state_vector=vector)

# Initialize reciprocal rotation module
reci = LookupRotation(negative_evals=eigs._negative_evals, evo_time=eigs._evo_time)
algo = HHL(matrix, vector, truncate_powerdim, truncate_hermitian, eigs,
           init_state, reci, num_q, num_a, orig_size)

result = algo.run(aqua_qinstance)
print(result.solution)

[ 0.66575607-5.39389682e-15j -0.38561455+4.35672537e-15j]

இப்போது:

தீர்வு திசையன் திரும்பப் பெறப்படவில்லை என்பதை நினைவில் கொள்க, ஏனெனில் அதற்கு தீர்வு சர்க்யூட்டின் அதிவேக விலையுயர்ந்த உருவகப்படுத்துதல் தேவைப்படும். அதற்கு பதிலாக, தீர்வில் காணக்கூடியவற்றை மதிப்பீடு செய்ய சுற்று பயன்படுத்தப்படலாம். விவரங்களுக்கு, HHL இன் ஆவணங்கள் மற்றும் ஆவணங்களை பார்க்கவும்.

கட்ட மதிப்பீடு¶

சுருக்கம்¶

கட்ட மதிப்பீடு முற்றிலும் மறுசீரமைக்கப்பட்டுள்ளது, மேலும் ஒரு ஹாமில்டோனியனின் சமநிலைக் கணக்கீட்டைப் பயன்படுத்தப் பயன்படுத்தப்பட்ட ஒரு qiskit.aqua.algorithms.QPE வகுப்பிற்குப் பதிலாக, இப்போது எங்களுக்கு இரண்டு தனித்தனி செயலாக்கங்கள் உள்ளன: HamiltonianPhaseEstimation பங்கு வகிக்கிறது பழைய QPE மற்றும் பாடநூல் கட்ட மதிப்பீட்டிற்கான புதிய PhaseEstimation வழிமுறை.

செயல்பாட்டு கட்ட மதிப்பீடு, qiskit.aqua.algorithms.IQPE இன்னும் மாற்றப்படவில்லை, ஆனால் விரைவில் பின்பற்றப்படும்.

இடம்பெயர்வு வழிகாட்டி¶

இன் சமத்துவ மதிப்பைக் கண்டுபிடிப்பதில் உள்ள சிக்கலைக் கருத்தில் கொள்வோம்

\[H = 0.5 X + Y + Z\]

உள்ளீட்டு நிலையில் \(|0\rangle\).

[47]:

state_in = np.array([1, 0])

முன்பு:

[48]:

from qiskit.circuit.library import QFT
from qiskit.aqua.algorithms import QPE
from qiskit.aqua.components.initial_states import Custom
from qiskit.aqua.operators import I, X, Y, Z

n_ancillae = 5
num_time_slices = 1

op = 0.5 * X + Y + Z

state_preparation = Custom(op.num_qubits, state_vector=state_in)
iqft = QFT(n_ancillae, do_swaps=False).inverse().reverse_bits()

qpe = QPE(op, state_preparation, iqft, num_time_slices, n_ancillae,
          expansion_mode='trotter',
          shallow_circuit_concat=True)


result = qpe.run(aqua_qinstance)

print(result.eigenvalue)

0.78125

புதியது:

[49]:

from qiskit import BasicAer

மேலும் எடுத்துக்காட்டுகள்¶

இப்போது நாம் தீர்க்க நிலையான கட்ட மதிப்பீட்டையும் செய்யலாம்

\[U|\psi\rangle = e^{2\pi i\phi}|\psi\rangle.\]
[50]:

from qiskit.circuit import QuantumCircuit
from qiskit.algorithms import PhaseEstimation

unitary = QuantumCircuit(1)
unitary.z(0)

state_in = QuantumCircuit(1)
state_in.x(0)  # eigenstate |1> with eigenvalue -1, hence a phase of phi = 0.5

pe = PhaseEstimation(num_evaluation_qubits=3, quantum_instance=qinstance)
result = pe.estimate(unitary, state_in)

print(result.most_likely_phase)

0.5

VQC இருப்பிடத்தை qiskit_machine_learning.algorithms.VQC என மாற்றியது, இப்போது இது பொதுவான மாறுபட்ட வழிமுறைக்கு பதிலாக NeuralNetworkClassification பொருளாக செயல்படுத்தப்படுகிறது. இடைமுகம் அதற்கேற்ப புதுப்பிக்கப்பட்டுள்ளது, கீழே உள்ள பயிற்சிகளைப் பார்க்கவும்.

EOH¶

from qiskit.aqua.algorithms import EOH
from qiskit.aqua.operators import WeightedPauliOperator
from qiskit.aqua.components.initial_states import Custom
from qiskit.quantum_info import Pauli

hamiltonian = WeightedPauliOperator([[1, Pauli('XX')], [1, Pauli('ZZ')], [1j, Pauli('YY')]])

observable = WeightedPauliOperator([[1, Pauli('XI')]])

initial_state = Custom(2, 'uniform')

evo_time = 2
num_time_slices = 10

eoh = EOH(observable, initial_state, hamiltonian, evo_time=evo_time, num_time_slices=num_time_slices)
result = eoh.run(aqua_qinstance)
result['avg']

(-0.6536436208636107+5.551115123125783e-17j)

புதியது:

[52]:

import numpy as np
from qiskit.opflow import I, X, Y, Z, PauliTrotterEvolution, StateFn

hamiltonian = (X ^ X) + (Z ^ Z) + 1j * (Y ^ Y)

observable = X ^ I

initial_state = StateFn(np.ones(4) / 2)

evo_time = 2
num_time_slices = 10

# get the evolution operator
evolved_hamiltonian = (evo_time * hamiltonian).exp_i()

evo = PauliTrotterEvolution(reps=num_time_slices)
evo.convert(evolved_hamiltonian)

# get the evolved state
evolved_state = evolved_hamiltonian @ initial_state

# evaluate the target observable at the evolved state
print((~StateFn(observable) @ evolved_state).eval())

(-0.653643620863615+0j)

அளவீட்டுப் பிழை குறைத்தல்¶

முன்னுரை¶

அளவீட்டு பிழைகள் குறைக்க அளவீட்டு அளவொப்புமை பயன்படுத்தப்படுகிறது. அனைத்து \(2^n\) அடிப்படை உள்ளீட்டு நிலைகளையும் தயார் செய்வதும், பிற அடிப்படை நிலைகளில் எண்ணிக்கையை அளவிடுவதற்கான நிகழ்தகவைக் கணக்கிடுவதும் முக்கிய யோசனை. இந்த அளவொப்புமை, ஆர்வத்தின் மற்றொரு பரிசோதனையின் சராசரி முடிவுகளையும் சரிசெய்ய முடியும். இந்த குறிப்பேடு ignis.mitigation.measurement தொகுதியை எவ்வாறு பயன்படுத்துவது என்பதற்கான எடுத்துக்காட்டுகளை வழங்குகிறது.

[1]:

# Import general libraries (needed for functions)
import numpy as np
import time

# Import Qiskit classes
import qiskit
from qiskit import QuantumRegister, QuantumCircuit, ClassicalRegister, Aer
from qiskit.providers.aer import noise
from qiskit.tools.visualization import plot_histogram

# Import measurement calibration functions
from qiskit.ignis.mitigation.measurement import (complete_meas_cal, tensored_meas_cal,
                                                 CompleteMeasFitter, TensoredMeasFitter)

அளவொப்புமை மெட்ரிக்ஸின் 3 க்யூபிட் எடுத்துக்காட்டு¶

5-க்யூபிட் குவாண்டம் பதிவேட்டில் [Q0, Q1, Q2, Q3, Q4], 3 க்யூபிட்ஸ் Q2, Q3 மற்றும் Q4 க்கான அளவொப்புமை மேட்ரிக்ஸை உருவாக்க விரும்புகிறோம் என்று வைத்துக் கொள்ளுங்கள்.

நம்மிடம் 3 க்யூபிட்ஸ் இருப்பதால், அதில் \(2^3=8\) சாத்தியமான குவாண்டம் நிலைகள் உள்ளன.

அளவீட்டு அளவொப்புமை சர்க்யூட்களை உருவாக்குதல்¶

முதலில், முழு ஹில்பர்ட் வெளியீட்டு அளவீட்டு அளவொப்புமை சர்கூட்களின் பட்டியலை உருவாக்குகிறோம். ஒவ்வொரு சர்கூட்டும் ஒரு அடிப்படை நிலையை உருவாக்குகிறது. இங்கு \(n=3\) க்யூபிட்ஸ் இருந்தால், நாம் \(2^3=8\) அளவொப்புமை சர்கூட்கள் பெறுகிறோம்.

பின்வரும் செயல்பாடு complete_meas_cal அளவொப்புமை சர்க்யூட்கள் கொண்ட QuantumCircuit பொருட்களின் meas_calibs மற்றும் அளவுத்திருத்த நிலை லேபிள்களின் நிலை_அடையாளக் குறியீடு ஆகியவற்றை வழங்குகிறது.

இந்த செயல்பாட்டிற்கான உள்ளீட்டை பின்வரும் மூன்று வடிவங்களில் ஒன்றில் கொடுக்கலாம்:

  • க்யூபிட்_list: அளவீட்டு திருத்தம் செய்ய க்யூபிட்ஸ்-களின் பட்டியல், அல்லது:

  • qr (QuantumRegister): ஒரு குவாண்டம் ரெஜிஸ்டர், அல்லது:

  • cr (ClassicalRegister): ஒரு கிளாசிக்கல் பதிவீடு.

கூடுதலாக, ஒருவர் ஒரு ஸ்ட்ரிங் circlabel-ஐ வழங்க முடியும், இது தனிப்பட்ட அடையாளங்களுக்காக சர்கூட் பெயர்களின் தொடக்கத்தில் சேர்க்கப்படுகிறது.

எடுத்துக்காட்டாக, நம் விஷயத்தில், உள்ளீடு என்பது 5-qubit QuantumRegister என்பது Q2, Q3, Q4 ஆகிய qubit-களைக் கொண்டுள்ளது:

[2]:

# Generate the calibration circuits

qubit_list = [2,3,4]
meas_calibs, state_labels = complete_meas_cal(qubit_list=qubit_list, qr=qr, circlabel='mcal')

அளவொப்புமை மேட்ரிக்ஸைக் கணக்கிடு¶

நாங்கள் எந்த இரைச்சலையும் பயன்படுத்தாவிட்டால், அளவொப்புமை மேட்ரிக்ஸ் \(8 \times 8\) அடையாளம் மேட்ரிக்ஸ் என்று எதிர்பார்க்கப்படுகிறது.

முடிவுகளை பகுப்பாய்வு செய்தல்¶

மொத்த அளவீட்டு சரிபார்த்தலையும், ஒரு குறிப்பிட்ட க்யூபிட்-ற்கான அளவீட்டு சரிபார்த்தலையும் கணக்கிட விரும்புகிறோம், எடுத்துக்காட்டாக, Q0.

அளவுத்திருத்த மேட்ரிக்ஸின் மேல்-மூலைவிட்ட கூறுகள் நிலை ‘x’, அளவிடப்பட்ட நிலை ‘x’ ஐ அளவிடுவதற்கான நிகழ்தகவுகள் என்பதால், இந்த மேட்ரிக்ஸின் சுவடு சராசரி ஒதுக்கீட்டு நம்பகத்தன்மையாகும்.

அளவொப்புயை பயன்படுத்துதல்¶

நாம் இப்போது மற்றொரு பரிசோதனையைச் செய்து அளவிடப்பட்ட முடிவுகளை சரிசெய்கிறோம்.

3Q GHZ நிலையில் சரியான அளவீட்டு இரைச்சல்¶

உதாரணமாக, Q2, Q3, Q4 என்ற க்யூபிட்ஸ்-களில் 3-க்யூபிட் GHZ நிலையில் தொடங்குவோம்:

\[\mid GHZ \rangle = \frac{\mid{000} \rangle + \mid{111} \rangle}{\sqrt{2}}\]
[11]:

# Make a 3Q GHZ state
cr = ClassicalRegister(3)
ghz = QuantumCircuit(qr, cr)
ghz.h(qr[2])
ghz.cx(qr[2], qr[3])
ghz.cx(qr[3], qr[4])
ghz.measure(qr[2],cr[0])
ghz.measure(qr[3],cr[1])
ghz.measure(qr[4],cr[2])

[11]:

<qiskit.circuit.instructionset.InstructionSet at 0x7f869dce3040>

நாம் இப்போது அளவொப்புமை சர்கூட்களை இயக்குகிறோம் (மேலே உள்ள இரைச்சல் மாதிரியுடன்):

[12]:

job = qiskit.execute([ghz], backend=backend, shots=5000, noise_model=noise_model)
results = job.result()

எந்தவொரு பிழை குறைத்தலும் இல்லாமல் மற்றும் தணிப்புடன் முடிவுகளை இப்போது கணக்கிடுகிறோம், அதாவது முடிவுகளுக்கு அளவொப்புமை மேட்ரிக்ஸைப் பயன்படுத்திய பிறகு.

அளவொப்புயை பயன்படுத்துவதற்கு இரண்டு பொருத்தமான முறைகள் உள்ளன (எந்த முறையும் வரையறுக்கப்படவில்லை என்றால், ‘least_squares’ பயன்படுத்தப்படுகின்றன).- ‘pseudo_inverse’, இது அளவொப்புமை மேட்ரிக்ஸின் நேரடி தலைகீழ் ஆகும், - ‘least_squares’, இது உடல் நிகழ்தகவுகளைக் கட்டுப்படுத்துகிறது.

சரிசெய்ய வேண்டிய பதனிடப்படாத தரவுகளை பல வடிவங்களில் கொடுக்கலாம்:

  • வடிவம்1: results.get_counts -ல் இருந்து ஒரு எண்ணும் அகராதி,

  • வடிவம்2: length=len(state_labels) எண்ணிக்கையின் பட்டியல்,

  • வடிவம்3: length=M*len(state_labels) எண்ணிக்கையின் பட்டியல், அதில் M என்பது ஒரு integer (எ.கா. டோமோகிராஃபி தரவுடன் பயன்படுத்த),

  • வடிவம்4: ஒரு qiskit முடிவு (எ.கா. மேலே உள்ள முடிவுகள்).

[13]:

# Results without mitigation
raw_counts = results.get_counts()

# Get the filter object
meas_filter = meas_fitter.filter

# Results with mitigation
mitigated_results = meas_filter.apply(results)
mitigated_counts = mitigated_results.get_counts(0)

இப்போது நாம் நம் முடிவுககளை பிழை தணித்தும் தணிக்காமலும் வகுக்கலாம்:

[14]:

டெண்சர்ட் குறைத்தல்¶

பிழை லோக்கல் என்று தெரிந்தால் அளவுத்திருத்தத்தை எளிமைப்படுத்தலாம். “local error” என்பதன் மூலம் பிழையை க்யூபிட்ஸ்-களின் துணைக்குழுக்களுக்கு மாற்றலாம். இந்த வழக்கில், அளவொப்புமை மேட்ரிக்ஸின் கணக்கீட்டிற்கு \(2^n\) க்கும் குறைவான நிலைகளே தேவைப்படுகின்றன.

பிழை க்யூபிட் 2 மற்றும் க்யூபிட்ஸ் 3 மற்றும் 4 இல் லோக்கலில் செயல்படுகிறது என்று வைத்துக் கொள்ளுங்கள். tensored_meas_cal செயல்பாட்டைப் பயன்படுத்தி அளவொப்புமை சர்கூட்களை உருவாக்கவும். முன்பு போலல்லாமல், க்யூபிட் பட்டியலை வெளிப்படையாக துணைக்குழு பகுதிகளாகப் பிரிக்க வேண்டும்.

உருவாக்கப்பட்ட சர்கூட்களின் பெயர்களை இப்போது மீட்டெடுக்கிறோம். ஒவ்வொரு லேபிளிலும் (நீளம் 3), குறைவான குறிப்பிடத்தக்க பிட், க்யூபிட் 2 உடன் ஒத்துப்போகிறது, நடுத்தர பிட், க்யூபிட் 3 உடன் ஒத்திருக்கிறது, மற்றும் மிக முக்கியமான பிட், க்யூபிட் 4 உடன் ஒத்துள்ளது என்பதை நினைவில் கொள்க.

சர்கூட் பெயர்களை விரிவாகக் கூறுவோம். எட்டுக்கு பதிலாக நான்கு சுற்றுகள் மட்டுமே இருப்பதைக் காண்கிறோம். தேவையான சுற்றுகளின் மொத்த எண்ணிக்கை:math: 2 ^m எங்கே:math:m என்பது இலக்கு துணைக்குழுவில் உள்ள குவிட்களின் எண்ணிக்கை (இங்கே:math: m = 2).

க்யூபிட்ஸ் 3 மற்றும் 4 இன் ஒவ்வொரு அடிப்படை நிலையும் சரியாக ஒரு முறை தோன்றும். க்யூபிட் 2 க்கு இரண்டு அடிப்படை நிலைகள் மட்டுமே தேவை, எனவே இவை நான்கு சோதனைகளிலும் சமமாக பிரிக்கப்படுகின்றன. எடுத்துக்காட்டாக, க்யூபிட் 2 இன் நிலை ‘0’ ஆக இருப்பின், நிலை அடையாளக் குறியீடு ‘000’ மற்றும் ‘010’ இல் தோன்றும்.

முன்பு இருந்த அதே இரைச்சல் மாதிரியைப் பயன்படுத்தி, இப்போது Aer சிமுலேட்டர்-ல் அளவொப்புமை சர்கூட்களை இயக்குகிறோம். இந்த இரைச்சல் உண்மையில் க்யூபிட்ஸ் 3 மற்றும் 4 க்கு தனித்தனியாக உள்ளது, ஆனால் அது நமக்குத் தெரியாது என்று வைத்துக் கொள்வோம், மேலும் இது க்யூபிட் 2 க்கு லோக்கல் என்று மட்டுமே நமக்குத் தெரியும்.

அளவீட்டு பிழை குறைப்புடன் Qiskit வழிமுறைகளை இயக்குகிறது¶

ஒரு Qiskit வழிமுறையின் ஒரு பகுதியாக குவாண்டம் குவாண்டம் சர்கூட்களை இயக்கும் போது அளவீட்டு பிழை குறைத்தலைப் பயன்படுத்த, குவாண்டம் உதாரணம் அந்தந்த அளவீட்டு பிழை பொருத்தும் உதாரணத்தை நாம் சேர்க்க வேண்டும். இந்த பொருள் தேர்ந்தெடுக்கப்பட்ட பின்தளத்தில் விவரக்குறிப்புகளையும் கொண்டுள்ளது.

பின்வருவனவற்றில், VQE உடன் ஒரு ஹாமில்டோனியனின் அடி நிலையைத் தேடுவதற்கான எடுத்துக்காட்டில் Aqua வழிமுறைகளின் அளவீட்டு பிழை குறைத்தல்.

முதலில், பின்தளத்தில் வழங்கும் libraries மற்றும் வழிமுறையை இயக்கத் தேவையான வகுப்பு இறக்குமதி செய்ய வேண்டும்.

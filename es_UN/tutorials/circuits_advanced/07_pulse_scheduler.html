Usando el Planificador¶

El planificador traducirá un QuantumCircuit en un Schedule de pulsos, utilizando calibraciones de compuertas y mediciones: una descripción optimizada a nivel de pulsos de una operación en qubits particulares.

Los backends que están habilitados para OpenPulse normalmente tendrán calibraciones definidas para las mediciones y para cada una de sus compuertas base. El usuario también puede definir o actualizar las calibraciones.

Uso básico¶

Para empezar, construye un circuito cuántico como lo harías normalmente. Para nuestro ejemplo a continuación, creamos un estado de Bell simple.

[1]:

from qiskit import QuantumCircuit

circ = QuantumCircuit(2, 2)
circ.h(0)
circ.cx(0, 1)
circ.measure([0, 1], [0, 1])

circ.draw('mpl')

[1]:

Usaremos el backend emulado, FakeAlmaden, para demostrar cómo usar el planificador. Contiene calibraciones predeterminadas para la medición y para sus compuertas base. La operación Hadamard no es una de esas compuertas base, por lo que usamos el transpilador para compilar nuestro circuito de entrada en un circuito equivalente en términos de las compuertas base del dispositivo.

[2]:

from qiskit import transpile, schedule as build_schedule
from qiskit.test.mock import FakeAlmaden

backend = FakeAlmaden()

transpiled_circ = transpile(circ, backend)  # Undefined Hadamard is replaced by U1
schedule = build_schedule(transpiled_circ, backend)

Veamos cómo se ve nuestro planificador schedule construido a partir del circuito transpiled_circ.

[3]:

schedule.draw(plot_range=[0, 2000])

[3]:

¡Eso cubre lo básico! Usamos el transpilador para reescribir el circuito en términos de las compuertas base y luego usamos las calibraciones predeterminadas del backend para programar el circuito transpilado. A continuación, pasaremos por la planificación con calibraciones que construiremos nosotros mismos.

Planificación con definiciones de compuertas personalizadas¶

Si tu circuito de entrada tiene calibraciones definidas, utilizará esas calibraciones al planificar tu circuito.

[4]:

from qiskit import pulse

with pulse.build() as h_q0:
    pulse.play(pulse.library.Gaussian(duration=256, amp=0.2, sigma=50, name='custom'),
               pulse.DriveChannel(0))

circ.add_calibration('h', [0], h_q0)

schedule = build_schedule(circ, backend)
schedule.draw(plot_range=[0, 2000])

[4]:

Observa que el pulso inicial en D0, la compuerta Hadamard, ahora se implementa con nuestro pulso personalizado.

Métodos del planificador¶

El planificador puede seguir varias rutinas. De forma predeterminada, sigue una regla lo más tarde posible (as late as possible, ALAP). Otro método de planificación es lo antes posible, (as soon as possible, ASAP). Para ambos métodos, la planificación de salida es mínima: en la ruta de operación de mayor duración del circuito de entrada, la hora de inicio de cada operación es el tiempo de finalización de la operación en curso. Los métodos determinan cómo planificar operaciones fuera de esa ruta más larga.

Esto queda claro a través de un ejemplo:

[5]:

circ = QuantumCircuit(2, 2)
circ.x(0)
circ.x(0)
circ.x(1)
circ.measure([0, 1], [0, 1])

[5]:

<qiskit.circuit.instructionset.InstructionSet at 0x12cba98e0>

Para nuestro backend emulado, estas operaciones de compuerta \(X\) tienen la misma duración (duration = 160 dt). Para que la planificación sea mínima, las dos operaciones \(X\) en el qubit 0 se planificarán una tras otra, y el pulso de medición seguirá inmediatamente al segundo de esos pulsos.

ALAP elegirá el último tiempo válido para planificar operaciones con menos restricciones, por lo que la compuerta \(X\) en el qubit 1 se reproducirá en sincronía con la segunda compuerta :math:`X en el qubit 0.

[6]:

schedule = build_schedule(circ, backend,
                          method="as_late_as_possible")
schedule.draw(channels=[pulse.DriveChannel(0), pulse.DriveChannel(1)])

[6]:

Por otro lado, como sugiere el nombre, ASAP planificará operaciones tan pronto como sus recursos estén libres. Por lo tanto, la compuerta \(X\) en el qubit 1 se planificará en time=0, en sincronía con la primera compuerta \(X\) en el qubit 0.

[7]:

schedule = build_schedule(circ, backend,
                          method="as_soon_as_possible")
schedule.draw(channels=[pulse.DriveChannel(0), pulse.DriveChannel(1)])

[7]:

ALAP es el predeterminado porque permite que los qubits permanezcan inactivos el mayor tiempo posible. En este caso, la diferencia entre ALAP y ASAP puede ser insignificante, pero en ALAP, el qubit 0 no tiene tiempo para decaer desde estado excitado antes de la medición.

Instalando Qiskit¶

Requisitos¶

Qiskit supports Python 3.6 or later. However, both Python and Qiskit are evolving ecosystems, and sometimes when new releases occur in one or the other, there can be problems with compatibility.

Si tienes problemas instalando o usando Qiskit después de actualizar Python, revisa Qiskit Package metadata for Programming Language para ver qué versiones específicas de Python son soportadas.

Recomendamos instalar Anaconda, una distribución Python multiplataforma para el cálculo científico. A su vez, es recomendable el uso de Jupyter, incluido en Anaconda, para trabajar con Qiskit.

Qiskit está probado y soportado en los siguientes sistemas de 64-bit:

  • Ubuntu 16.04 o posterior

  • macOS 10.12.6 o posterior

  • Windows 7 o posterior

Instalación¶

Recomendamos utilizar entornos virtuales de Python para separar claramente Qiskit de otras aplicaciones y mejorar su experiencia.

La forma más simple de usar los entornos es usando el comando conda, que se incluye con Anaconda. Un entorno Conda te permite especificar una versión específica de Python y un conjunto de librerías. Abre una ventana de terminal en la carpeta donde quieres trabajar.

It is preferred that you use Anaconda prompt installed with the Anaconda. All you have to do is create a virtual environment inside Anaconda and activate the environment. These commands can be run in Anaconda prompt irrespective of Windows or Linux machine.

Crea un entorno mínimo sólo con Python instalado.

conda create -n name_of_my_env python=3

Activate your new environment.

conda activate name_of_my_env

Nota

If you are using conda versions prior to 4.6, use: For Linux or macOS: source activate name_of_my_env For Windows: activate name_of_my_env

Después, instala el paquete de Qiskit, que incluye Terra, Aer, Ignir y Aqua.

pip install qiskit

Nota

Starting with Qiskit 0.13.0 pip 19 or newer is needed to install qiskit-aer from the precompiled binary on Linux. If you do not have pip 19 installed you can run pip install -U pip to upgrade it. Without pip 19 or newer this command will attempt to install qiskit-aer from sdist (source distribution) which will try to compile aer locally under the covers.

If the packages were installed correctly, you can run conda list to see the active packages in your virtual environment.

If you intend to use visualization functionality or Jupyter notebooks it is recommended to install qiskit with the visualization extra requirements

pip install qiskit[visualization]

It is worth pointing out that if you’re a zsh user (which is the default shell on newer versions of macOS), you’ll need to put qiskit[visualization] in quotes:

pip install 'qiskit[visualization]'

Nota

Después de haber instalado y verificado los paquetes de Qiskit que quieres usar, impórtalos en tu entorno con Python para comenzar a trabajar.

import qiskit

Nota

If you want to contribute to the Qiskit community by developing and contributing code with the most recently updated Qiskit code, see Build Qiskit packages from source.

Accede a los Sistemas Cuánticos de IBM¶

IBM Quantum ofrece varios ordenadores cuánticos reales y simuladores cuánticos de alto rendimiento para ordenadores clásicos a través de la IBM Quantum Experience con Qiskit. Sigue los siguientes pasos para configurar el entorno en Qiskit y enviar los trabajos a los sistemas cuánticos de IBM.

Nota

With the release of Qiskit 0.11, if you had previously saved your IBM Quantum Experience credentials locally, you might need to update them so that you can use the new IBM Quantum Experience V2. See Updating your IBM Quantum Experience Credentials.

To configure your account, you will create a local configuration file which includes an API key

1. Crea una cuenta gratis de IBM Quantum Experience.

2. Navigate to My Account to view your account settings.

3. Click on Copy token to copy the token to your clipboard. Temporarily paste this API token into your favorite text editor so you can use it later to create an account configuration file.

4. Run the following commands to store your API token locally for later use in a configuration file called qiskitrc. Replace MY_API_TOKEN with the API token value that you stored in your text editor.

from qiskit import IBMQ
IBMQ.save_account('MY_API_TOKEN')

Si buscas más detalles, por ejemplo sobre cómo gestionar varias credenciales de cuentas de IBM Quantum, puedes visitar el tutorial The IBM Quantum Account.

Comprobar la Versión Instalada¶

Since the Qiskit package includes a constellation of different elements, simply printing the version by running qiskit.__version__ can be misleading as it returns only the version for the qiskit-terra package. This is because the qiskit namespace in Python doesn’t come from the Qiskit package, but instead is part of the qiskit-terra package.

import qiskit
qiskit.__version__

'0.16.0'

Para ver las versiones de todos los elementos de Qiskit de tu entorno puedes usar el atributo __qiskit_version__. Por ejemplo, ejecutar el siguiente comando te devolverá un diccionario que incluye las versiones de cada uno de los paquetes de Qiskit instalados.

qiskit.__qiskit_version__

{'qiskit-terra': '0.16.0',
 'qiskit-aer': '0.7.0',
 'qiskit-ignis': '0.5.0',
 'qiskit-ibmq-provider': '0.11.0',
 'qiskit-aqua': '0.8.0',
 'qiskit': '0.23.0'}

Truco

Si estás creando una incidencia o necesitas compartir por cualquier motivo las versiones de Qiskit que tienes instaladas, usa el atributo __qiskit_version__.
